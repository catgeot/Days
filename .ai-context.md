# gateo.kr — AI 인수인계 컨텍스트

**최종 갱신**: 2026-07-24  
**대상**: Cursor 등 **AI 에이전트** (사람이 매번 읽는 문서가 아님)  
**역할**: 새 세션·새 모델이 **코드 수정·운영 판단**에 필요한 **지속 규칙·SSOT·현재 상태**만 담는다.

## 0. 사용법 (필독)

### Cursor에서 이 파일을 읽는 방법

| 방식 | 동작 |
|------|------|
| **`@.ai-context.md` 연결** | 파일 내용이 대화에 **직접 주입** (가장 확실) |
| **프로젝트 Rule** | [`.cursor/rules/gateo-project-context.mdc`](.cursor/rules/gateo-project-context.mdc) (`alwaysApply: true`) — 에이전트에게 **읽으라고 지시** (Read 도구로 본문 로드) |
| **언급·Rule 없음** | 맥락 **없음** |
| **세션·대화 간** | 주입·읽기 기록 **유지 안 됨** — 큰 작업마다 `.ai-context` 재확인 |

**User Rule**(Cursor 설정)은 저장소 밖. commit·릴리스 노트는 User Rule + 본 파일 **1.5.1** / **1.7**이 SSOT.

### 이 파일 vs 다른 문서

| 문서 | 담는 것 | 갱신 주체 |
|------|---------|-----------|
| **`.ai-context.md`** | 불변에 가까운 **규칙·구조·운영 명령·미완 1줄** | AI, 세션 종료 시 **최소** |
| **`plans/YYYY-MM-DD-project-log.md`** | 그날 한 **완료·커밋·QA·결정** | AI, 2~5줄/일 |
| **`plans/*.md` (운영 가이드)** | 공항·페리·여행지 추가 등 **절차·스키마** | 기능 변경 시 |
| **`src/data/releaseNotes.js`** | 사용자-facing **업데이트 공지** SSOT · `SiteUpdateBanner` · FooterModal Updates |
| **git / `plans/archive/`** | 장문 히스토리·구 계획서 | 거의 갱신 안 함 |

**원칙**: 완료 보고·커밋 해시·단계별 타임라인은 **일지**에만. 여기에 **복붙 금지**.

## 1. 유지 규약 (AI·세션 종료 시)

### 1.1 목적

- 사용자 인수인계서가 아니라 **다음 AI 세션을 위한 압축 메모리**.
- 「무엇을 했나」보다 **「지금 무엇이 맞고, 무엇을 하면 안 되나」**.

### 1.2 분량

- **목표 ~200줄, 상한 ~250줄**. 넘으면 **5절·6절**부터 잘라낸다.
- 한 항목은 **1~3문장**. 표·체크리스트·npm 명령은 6절에만 두어도 됨.

### 1.3 절별 — 넣을 것 / 넣지 말 것

| 절 | 넣을 것 | 넣지 말 것 |
|----|---------|------------|
| **0** | 사용법·자동 로드 여부 | — |
| **1** | 이 규약 | 작업 내역 |
| **2~3** | 스택·폴더·**장애 방지** 규칙 | 완료된 기능 설명 장문 |
| **4** | 협업·커밋·UI 변경 금지 등 | — |
| **5** | **현재 스냅샷** 5~10줄, **미완·리스크**만 | 완료 로그·SHA 나열·Phase 타임라인 |
| **6** | **남은 과제**·운영 SSOT·npm·체크리스트 | 완료된 Phase 상세 |
| **7** | 문서·일지 루틴 | — |

### 1.4 갱신 시점

**갱신 O**

- 아키텍처·SSOT 파일·우선순위가 바뀜 (예: slug-first, Mapbox PROD 고정).
- **미완**·**배포 대기**·**재배포 필요**가 생기거나 해소됨.
- 새 에이전트가 **같은 실수를 반복**할 만한 규칙이 생김 (3절에 추가).

**갱신 X** (일지만)

- 버그 수정·UI 미세 조정·단일 커밋 완료.
- 「완료·배포됨·QA 통과」만 있는 항목 → 5절에서 **삭제**하고 일지에만 남김.

### 1.5 세션 종료 체크리스트 (AI)

1. `git status` — **커밋 게이트**(아래 **1.5.1**). 통과 시 **한글** 메시지로 commit(요청 불필요). 미통과면 커밋·푸시 금지.
2. `plans/YYYY-MM-DD-project-log.md` — **오늘 2~5줄** (없으면 새 파일, 직전 일지 링크).
3. **릴리스 노트** — 아래 **1.7**. 기본은 **생략**. 새 기능·중대 업데이트일 때만 초안 제안.
4. `.ai-context.md` — **필요할 때만**: 상단 날짜, **5절 3줄 이내**, **6절** 미완/완료 1줄 조정.
5. **새 `plans/*-완료보고.md` 생성 금지** — 사용자가 명시 요청한 경우만.

### 1.5.1 커밋·푸시 게이트 (의도)

예전 「사용자 요청 시에만 commit」은 **스모크·테스트 없이 깨진 로직을 커밋·푸시하던 사고**를 막기 위한 것이었다.  
게이트는 **요청 여부**가 아니라 **검증·이상 없음**이다. Cursor User Rule「Git commit/push — verification gate」와 동일.

| | 허용 | 금지 |
|--|------|------|
| **커밋** | 관련 audit/스모크/테스트 **PASS** · 에이전트가 아는 깨짐 없음 · 메시지 **한글** | 검증 생략 · FAIL/미확인 상태로 커밋 |
| **푸시** | 위와 동일 + 보통 **feature 브랜치** · Cloud 오케스트레이터는 method **§3.4**(PR 포함) | `main` 직접 push · force-push to main · FAIL tip/코드 푸시 |
| **완료 단정** | (별개) 사람 QA·승인이 필요한 UI는 「제안/대기」 — 커밋과 혼동 금지 | 검증만으로 「PROD 완료」 단정 |

### 1.6 금지

- 5절에 **이력 누적** (파일 자체가 「쌓지 말 것」을 규정함).
- 일지·handoff 문서 **전체 복사**.
- 사용자 **테스트·승인 전** 「완료」 단정 (4절).
- 릴리스 노트 **합의 없이** [`releaseNotes.js`](src/data/releaseNotes.js) 반영 · **사소한 변경마다 초안 제안**.

### 1.7 릴리스 노트 (희소 · 합의 후)

**SSOT**: [`src/data/releaseNotes.js`](src/data/releaseNotes.js) — `SiteUpdateBanner`(홈 팝업) · FooterModal **Updates** 탭.  
Cursor User Rule「Release notes — major features only」와 동일.

**제안 O** (세션 종료·배포 전 **이때만** 초안 제시)

- **새 기능** (사용자가 새로 쓸 수 있는 능력)
- **중대한 기능/UX 업데이트** (기존 흐름이 눈에 띄게 바뀌거나 가치가 큰 개선)
- **제휴 입점·공지** (`partner` / `notice`) — 사용자에게 알려야 할 때

**제안 X** (묻지 않음 · 파일 미수정 · 일지에만)

- 버그픽스 · 미세 UI · 문구/스타일 조정
- 내부 SSOT/데이터 채움 · 오케스트레이터 tip · 리팩터 · CI/스모크 · 문서만
- 이미 공지한 기능의 **작은 후속** 조정

**절차** (해당될 때만 · **합의 후** 반영):

1. AI가 대화에 **초안만** 제시 — `id`, `category`, `title`, `items`. 「넣을까요?」한 번.
2. 사용자 **승인·수정 합의** 후에만 파일 추가 (`RELEASE_NOTES` **배열 맨 앞**).
3. `id`(보통 `YYYY-MM-DD`)가 바뀌면 「다시 보지 않기」 사용자에게도 팝업 재표시 — **잦은 id 남발 금지**.

**category**: `feature` · `partner` · `fix` · `notice` — `fix`는 **사용자 체감이 큰** 수정만(남발 금지).
