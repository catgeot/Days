# gateo.kr — AI 인수인계 컨텍스트

**최종 갱신**: 2026-05-22  
**대상**: Cursor 등 **AI 에이전트** (사람이 매번 읽는 문서가 아님)  
**역할**: 새 세션·새 모델이 **코드 수정·운영 판단**에 필요한 **지속 규칙·SSOT·현재 상태**만 담는다.

## 0. 사용법 (필독)

### Cursor에서 이 파일을 읽는 방법

| 방식 | 동작 |
|------|------|
| **`@.ai-context.md` 연결** | 파일 내용이 대화에 **직접 주입** (가장 확실) |
| **프로젝트 Rule** | [`.cursor/rules/gateo-project-context.mdc`](.cursor/rules/gateo-project-context.mdc) (`alwaysApply: true`) — 에이전트에게 **읽으라고 지시** (Read 도구로 본문 로드) |
| **언급·Rule 없음** | 맥락 **없음** |
| **세션·대화 간** | 주입·읽기 기록 **유지 안 됨** — 큰 작업마다 `.ai-context` 재확인 |

**User Rule**(Cursor 설정)은 저장소 밖 — commit·PR·문체 등 **전역** 규칙만 자동 적용.

### 이 파일 vs 다른 문서

| 문서 | 담는 것 | 갱신 주체 |
|------|---------|-----------|
| **`.ai-context.md`** | 불변에 가까운 **규칙·구조·운영 명령·미완 1줄** | AI, 세션 종료 시 **최소** |
| **`plans/YYYY-MM-DD-project-log.md`** | 그날 한 **완료·커밋·QA·결정** | AI, 2~5줄/일 |
| **`plans/*.md` (운영 가이드)** | 공항·페리·여행지 추가 등 **절차·스키마** | 기능 변경 시 |
| **git / `plans/archive/`** | 장문 히스토리·구 계획서 | 거의 갱신 안 함 |

**원칙**: 완료 보고·커밋 해시·단계별 타임라인은 **일지**에만. 여기에 **복붙 금지**.

## 1. 유지 규약 (AI·세션 종료 시)

### 1.1 목적

- 사용자 인수인계서가 아니라 **다음 AI 세션을 위한 압축 메모리**.
- 「무엇을 했나」보다 **「지금 무엇이 맞고, 무엇을 하면 안 되나」**.

### 1.2 분량

- **목표 ~200줄, 상한 ~250줄**. 넘으면 **5절·6절**부터 잘라낸다.
- 한 항목은 **1~3문장**. 표·체크리스트·npm 명령은 6절에만 두어도 됨.

### 1.3 절별 — 넣을 것 / 넣지 말 것

| 절 | 넣을 것 | 넣지 말 것 |
|----|---------|------------|
| **0** | 사용법·자동 로드 여부 | — |
| **1** | 이 규약 | 작업 내역 |
| **2~3** | 스택·폴더·**장애 방지** 규칙 | 완료된 기능 설명 장문 |
| **4** | 협업·커밋·UI 변경 금지 등 | — |
| **5** | **현재 스냅샷** 5~10줄, **미완·리스크**만 | 완료 로그·SHA 나열·Phase 타임라인 |
| **6** | **남은 과제**·운영 SSOT·npm·체크리스트 | 완료된 Phase 상세 |
| **7** | 문서·일지 루틴 | — |

### 1.4 갱신 시점

**갱신 O**

- 아키텍처·SSOT 파일·우선순위가 바뀜 (예: slug-first, Mapbox PROD 고정).
- **미완**·**배포 대기**·**재배포 필요**가 생기거나 해소됨.
- 새 에이전트가 **같은 실수를 반복**할 만한 규칙이 생김 (3절에 추가).

**갱신 X** (일지만)

- 버그 수정·UI 미세 조정·단일 커밋 완료.
- 「완료·배포됨·QA 통과」만 있는 항목 → 5절에서 **삭제**하고 일지에만 남김.

### 1.5 세션 종료 체크리스트 (AI)

1. `git status` — 사용자가 commit 요청 시에만 **한글** 메시지로 commit.
2. `plans/YYYY-MM-DD-project-log.md` — **오늘 2~5줄** (없으면 새 파일, 직전 일지 링크).
3. `.ai-context.md` — **필요할 때만**: 상단 날짜, **5절 3줄 이내**, **6절** 미완/완료 1줄 조정.
4. **새 `plans/*-완료보고.md` 생성 금지** — 사용자가 명시 요청한 경우만.

### 1.6 금지

- 5절에 **이력 누적** (파일 자체가 「쌓지 말 것」을 규정함).
- 일지·handoff 문서 **전체 복사**.
- 사용자 **테스트·승인 전** 「완료」 단정 (4절).

**Edge 배포 (401 방지)**: `VITE_SUPABASE_URL` subdomain = project ref 확인 후  
`npx supabase functions deploy <name> --project-ref <ref> --no-verify-jwt`  
(무백 JWT 호출 시 `--no-verify-jwt` 필요한 경우 많음 — 역할 변경 시 재점검.)

## 4. AI 협업 원칙

1. **기획 .md 남발 금지** — 사용자 명시 없으면 `plans/`에 새 완료보고/대형 계획서 만들지 않음. 일지 또는 본 파일 5~6절만.
2. **구조 합의 → 워크스페이스 직접 편집** — 채팅에 수백 줄 덤프 지양.
3. **UI 임의 변경 금지** — 레이아웃·반응형·모바일 승인 없이 변경 안 함. 한국어 `break-keep` 등 기존 규칙 유지.
4. **완료 단정** — 사용자 테스트·승인 없으면 「제안/대기」. commit은 **제안**만 (요청 시 실행).
5. **고유명사**: `한글명('English name')`.
6. **도메인 로직** (툴킷·갤러리·제휴): 3절·6절·해당 소스·Edge Function 우선.
7. **Git 커밋 메시지**: **한글** (Conventional prefix는 선택).

**런타임 우선순위**: 수동 오버라이드 → live `essential_guide` → toolkit-sync JSON → runtime-infer → multi-rule·좌표.

```powershell
cd c:\dev\days
npm run sync:airports-from-toolkit   # DB 읽기만, 툴킷 반영
npm run generate:airports            # spots 재생성, placeIds 유지
npm run audit:airports
```

**`generate:airports` 우선순위**: overrides.mjs → JSON `curated-override`(overrides에 slug 없을 때) → `toolkit-sync` → infer. **`spots` 직접 수정 금지** — overrides 후 generate.

| 기대 | |
|------|--|
| audit | `none: 0` |
| generate | `missing: 0`, drift Warning 확인 |
