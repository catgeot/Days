## 1. 프로젝트 개요
- **서비스명**: gateo.kr
- **설명**: React 기반의 모던 웹 애플리케이션으로, Supabase를 백엔드로 사용하며 3D 시각화 및 AI 기능을 통합한 서비스입니다.

## 2. 주요 기술 스택
- **Frontend**: React 19.2.0, Vite, React Router DOM
- **Styling**: Tailwind CSS, Lucide React
- **Backend & BaaS**: Supabase (`@supabase/supabase-js`)
- **3D & Data Visualization**: `react-globe.gl`, `three`, `@react-three/fiber`, `@react-three/drei`, `mapbox-gl`, `react-map-gl`
- **AI Integration**: `openai`, `@google/generative-ai`
- **State Management**: React Context API
- **Deployment**: Vercel

## 3. 폴더 구조 및 아키텍처
- `src/components/`: 재사용 가능한 UI 컴포넌트
- `src/pages/`: 라우팅되는 페이지 단위 컴포넌트
- `src/context/`: Context API를 활용한 전역 상태 관리 로직
- `src/shared/`: 공통 유틸리티, 커스텀 훅, 상수 등
- `supabase/`: Supabase 관련 설정, 데이터베이스 스키마 및 마이그레이션

### 🛠️ 주요 시스템 및 구조 설계 (참조 필수)
- **하이브리드 AI 모델 라우팅**: 위키는 Gemini 2.5 Pro, 툴킷은 Gemini 3.1 Pro (→2.5 Pro 폴백)로 목적에 맞춰 분리 구동.
- **지도 시스템**: Mapbox GL JS(`react-map-gl`) 적용. 3D 지형 효과(DEM) 및 Parallax UX.
- **갤러리 배경 연동**: `usePlaceGallery` 훅의 캐싱을 활용해 장소 검색 결과 모달 등의 배경으로 Unsplash 이미지가 딜레이 없이 로드됨.
- **SEO 및 라우팅 구조**: `/place/:slug` 형태의 딥링크 정적 라우팅이 적용. 동적 `sitemap.xml`(vite-plugin-sitemap), 캐노니컬 방어(Helmet), GA4 SPA(useLocation) 추적이 완비됨.

## 4. 🤖 AI 어시스턴트 핵심 행동 원칙 (Core Rules)
사용자의 요청에 따라 코드를 수정하거나 기능을 추가할 때 다음 프로세스를 **엄격하게 준수**합니다:
1. **논리적 구조 제안**: 수정 및 구현하려는 코드의 논리적 구조와 방향성을 먼저 설명합니다.
2. **사용자 승인**: 제안한 구조에 대해 사용자의 승인을 대기합니다.
3. **직접 파일 수정(Edit Tool 사용)**: 승인이 완료되면 채팅창에 전체 코드를 길게 출력하며 토큰을 낭비하는 대신, AI가 에디터의 파일을 직접 수정(Edit)하여 신속하게 변경사항을 반영합니다.
4. **디자인 임의 변경 금지 및 UI 밀도 원칙**: 모바일, 태블릿 대응 등 반응형 디자인 최적화를 포함하여 기존 UI/UX 디자인을 사용자의 명시적인 요청이나 합의 없이 일방적으로 변경하지 않습니다.
   - **스크롤 최소화 및 중요 버튼 누락 방지 원칙**: 장소 리뷰 등 작성하기/편집창 뷰를 구성할 때 전체 스크롤을 최소화(Scroll-Free)하여, 패딩/글자크기/버튼크기를 최적화.
   - **타이포그래피 및 단어 끊김 방지 원칙**: 한국어 텍스트가 렌더링되는 문단에는 반드시 CSS 속성(`word-break: keep-all;` 또는 Tailwind 클래스 `break-keep`)을 명시적으로 적용.
5. **백엔드 로직 수정 및 배포 의무**: Supabase Edge Function 수정 시 `npx supabase functions deploy <function_name> --project-ref <ID>` 명령어로 터미널을 통해 AI가 직접 배포를 우선 시도합니다.
6. **데이터 구조 및 Hydration 규칙 엄수**: DB에서 불러온 장소 데이터(destination)를 `PlaceCard` 컴포넌트에 넘길 때, **반드시 원본 DB 객체를 복사한 후 `name: target.destination` 등으로 매핑(Hydration)** 하여 빈 화면 에러를 방지합니다.
7. 🚨 **갤러리(이미지 검색/페이지네이션) 로직 수정 시 절대 주의**: API 검색어 최적화 시 영문 매핑(`FALLBACK_DICTIONARY`)을 함부로 제거하지 말고, 갱신(Refresh) 시 **기존 사진들에 새 사진들을 누적(Append)** 하는 방식을 유지해야 합니다.
8. **작업 단위 분할 및 커밋 주도 원칙 (Commit-Driven Workflow)**:
   - 🚨 **사용자 확인 필수 원칙 (절대 규칙)**: 코드 수정 후 반드시 사용자의 테스트 및 확인을 대기하며, 임의로 완료를 선언하지 않습니다.
   - 단위 작업 완료 시 `git commit -m "커밋 메시지"`를 제안.
9. **한글(영문) 고유명사 표기 규칙**: 장소나 상호명 생성 시 항상 `한글명('현지 영문명')` 형식을 엄수.
10. **계획 문서 작성 원칙 (Documentation Principle)**: 문서는 파일에 직접 기록하며 채팅창에 반복 출력하지 않습니다. "문서 작성 완료 + 2-3줄 요약"만 보고.

## 7. 🤖 Claude(Roo) 특화 설정

### 세션 종료 프로세스 (필수)
모든 세션 종료 전 반드시 수행:
1. **커밋 완료 확인**: `git commit -m "작업 내용 요약"`
2. **로그 파일 업데이트**: `plans/YYYY-MM-DD-project-log.md`에 오늘 작업 기록
3. **컨텍스트 파일 업데이트**: `.ai-context.md`의 "최근 수정 사항" 섹션 갱신 및 필요시 압축
4. **attempt_completion으로 종료**: `📌 다음 세션 시작 제시어: ".ai-context.md와 [로그 파일] 확인하고 [다음 작업]"` 제공

### 금지 패턴
- ❌ 일회성 보고서 파일 생성 (`-completion-report.md` 등 금지, 로그 파일에 누적)
- ❌ BEFORE/AFTER 코드 전체 복사 (토큰 낭비 방지)
- ❌ 작업 예상 시간 단정 (대신 작업 항목 나열 및 진행 로그 업데이트에 주력)
