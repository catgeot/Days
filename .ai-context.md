### 꼭 기억할 시스템 (토큰·장애 방지)

- **🚨 여행지 목록만 확인할 때**: **반드시** [`src/pages/Home/data/travelSpots-list.json`](src/pages/Home/data/travelSpots-list.json) (경량) 사용. [`travelSpots.js`](src/pages/Home/data/travelSpots.js)는 6,000+ 줄으로 **검토용 비권장**.  
  - 재생성: `node scripts/extract-travel-spots-list.cjs`
- **AI 모델**: 위키/툴킷 **역할 분리** (2.5 Pro / 3.1 Pro + 폴백). 프롬프트·키는 **에지 프록시** 쪽 정책을 따름.
- **Mapbox** 등: 토큰은 **URL 제한·서버/프록시** 관점의 보안 가이드 준수
- **SEO 라우트**: `/place/:slug` 및 `/place/:slug/:tab`, `vite-plugin-sitemap` + 동적 URL 목록
- **PlaceCard + Supabase/DB `destination`**: `PlaceCard`에 넣기 전 **객체 복사 후** `name` 등 **hydration** — 미매핑 시 **빈 UI**로 이어짐
- **갤러리** (`usePlaceGallery`): `FALLBACK_DICTIONARY` **임의 제거 금지**, 갱신 시 기존 사진에 **append** 유지. 장소 전환 시 **이전 API 응답이 늦게 덮어쓰지 않도록** 요청 세대·장소 키 기준 유지.

### Supabase Edge Function 배포 (401 방지)

배포 시: `VITE_SUPABASE_URL`의 **subdomain = project ref** 확인 후, 예)

`npx supabase functions deploy <function_name> --project-ref <ref> --no-verify-jwt`

(프론트에서 무백 JWT 호출이면 **401** 방지를 위해 `--no-verify-jwt` **필요**한 케이스가 많음. 보안·역할이 바뀌면 다시 점검할 것.)

## 4. AI 협업 핵심 원칙

1. **기획 .md 남발 금지**: 사용자가 **명시적으로** 요청하지 않는 한 `plans/*.md`에 **새 “완료 보고서/대형 계획”**을 만들지 않는다. 필요한 건 **짧은 일지** 또는 **이 파일 5~6절** 갱신.
2. **구조 합의 → 직접 편집**: 가능하면 채팅에 **수백 줄 코드 덤프** 대신, 워크스페이스 **파일을 수정**한다.
3. **UI 임의 변경 금지**: 기존 레이아웃/반응형/모바일 동작을 **승인 없이** 뜯지 않는다. 한국어 문단은 `break-keep` 등 **기존** 규칙 유지.
4. **사용자 확인**: 수정 후 “완료” **단정** 직전, 사용자 **테스트·승인**이 없으면 “제안/대기”로 둔다. 단위 작업 후 **git commit**은 **제안**한다.
5. **고유명사**: `한글명('English name')` 형식 유지.
6. **툴킷·갤러리·제휴**(스마트 링크, 비자/페리, 링크 정합 등) **도메인 로직**은 3절, 해당 **소스·Edge Function**을 **우선**한다.
