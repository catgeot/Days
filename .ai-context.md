# gateo.kr 프로젝트 컨텍스트 (AI Context)

## 1. 프로젝트 개요
- **서비스명**: gateo.kr
- **설명**: React 기반의 모던 웹 애플리케이션으로, Supabase를 백엔드로 사용하며 3D 시각화 및 AI 기능을 통합한 서비스입니다.

## 2. 주요 기술 스택
- **Frontend**: React 19.2.0, Vite, React Router DOM
- **Styling**: Tailwind CSS, Lucide React
- **Backend & BaaS**: Supabase (`@supabase/supabase-js`)
- **3D & Data Visualization**: `react-globe.gl`, `three`, `@react-three/fiber`, `@react-three/drei`
- **AI Integration**: `openai`, `@google/generative-ai`
- **State Management**: React Context API
- **Deployment**: Vercel

## 3. 폴더 구조 및 아키텍처
- `src/components/`: 재사용 가능한 UI 컴포넌트
- `src/pages/`: 라우팅되는 페이지 단위 컴포넌트
- `src/context/`: Context API를 활용한 전역 상태 관리 로직
- `src/shared/`: 공통 유틸리티, 커스텀 훅, 상수 등
- `supabase/`: Supabase 관련 설정, 데이터베이스 스키마 및 마이그레이션

## 4. 🤖 AI 어시스턴트 핵심 행동 원칙 (Core Rules)
사용자의 요청에 따라 코드를 수정하거나 기능을 추가할 때 다음 프로세스를 **엄격하게 준수**합니다:
1. **논리적 구조 제안**: 수정 및 구현하려는 코드의 논리적 구조와 방향성을 먼저 설명합니다.
2. **사용자 승인**: 제안한 구조에 대해 사용자의 승인을 대기합니다.
3. **직접 파일 수정(Edit Tool 사용)**: 승인이 완료되면 채팅창에 전체 코드를 길게 출력하며 토큰을 낭비하는 대신, AI가 에디터의 파일을 직접 수정(Edit)하여 신속하게 변경사항을 반영합니다.

## 5. 최근 수정 사항 및 남은 과제

### 진행 완료된 작업
- **장소 카드 모바일/반응형 UI 최적화 (진행 완료)**:
  - **헤더 1층/2층 반응형 재구성 (`PlaceChatPanel.jsx`)**: 가로 공간이 극히 좁은 모바일에서는 1층 가로 배치(`flex-row`)로 변경하여 세로 공간을 확보하고, 태블릿 환경에서는 2층(`flex-col`), 16인치 이상 대화면에서는 다시 1층 가로 배치로 동작하도록 반응형 레이아웃을 세밀하게 재조정했습니다. 모바일에서의 텍스트 잘림 현상을 막기 위해 텍스트 `truncate` 처리 및 버튼 크기를 미세 조정했습니다.
  - **모바일 스크롤 및 겹침 현상 해결 (`PlaceGalleryView.jsx`, `PlaceWikiDetailsView.jsx`)**: 모바일에서 상단 고정 헤더와 하단 고정 푸터(연관 여행지)가 컨텐츠를 가리는 문제를 해결하기 위해, 갤러리와 백과 컨테이너에 각각 상단 패딩(`pt-24`)과 하단 패딩(`pb-28~32`)을 넉넉히 주어 컨텐츠 영역을 완전히 분리했습니다.
  - **백과(Wiki) AI 버튼 모바일 접근성 개선 (`PlaceWikiDetailsView.jsx`)**: 기존에 최하단까지 스크롤해야만 볼 수 있었던 "제미나이에게 묻기" 버튼을 모바일 환경에서는 하단 고정 푸터(`fixed bottom-0`)로 분리하여 언제든 즉시 호출할 수 있도록 변경했습니다.
  - **유튜브 영상 추천 버튼 조건부 렌더링 (`YouTubePlayerView.jsx`)**: 모바일 영상 탭에서 동영상이 정상적으로 검색되었음에도 항상 노출되던 구글폼(영상 추천) 유도 버튼을, 동영상이 없을 때(`isEmpty`)만 나타나도록 렌더링 조건을 수정했습니다.
  - **영상 하이라이트(Moments) 텍스트 잘림 해결 (`VideoInfoView.jsx`)**: 영상 탭 좌측의 모먼트 설명이 두 줄 이상일 때 잘리던 문제(`truncate`, `max-w-150px`)를 해결하여, 긴 설명글도 온전히 줄바꿈(wrap)되어 읽을 수 있도록 레이아웃을 유연하게 개선했습니다. (유튜브 검색 API 자체의 `snippet.description` 텍스트 잘림 현상 제외)
- **UI 반응형 개선 (진행 완료)**: 장소 카드의 좌측 패널 헤더 디자인을 17인치 화면 기준으로 분기 처리하여 사용성을 크게 개선했습니다.
  - **헤더 2단 분리 (`PlaceChatPanel.jsx`)**: 모바일 및 16인치 이하 태블릿 환경(`xl` 브레이크포인트 미만)에서는 좁은 가로 공간을 효율적으로 사용하기 위해 상단에 '홈, 국가명, 여행지명'을 배치하고, 하단에 '펜툴(Log), 백과, 영상/갤러리' 버튼을 가로 중앙 정렬로 2층 배치했습니다. 17인치(`xl` 이상) 모니터에서는 기존대로 넓은 1단 가로 배치가 유지됩니다.
  - **`PLACE_OVERVIEW` 제거 (`GalleryInfoView.jsx`)**: 좁은 화면에서의 세로 공간 활용도를 높이기 위해, 사진 모드(`PHOTO_LOG`)가 아닐 때 표시되던 `PLACE_OVERVIEW` 구분선 영역을 17인치 미만 환경에서 숨김 처리(`hidden xl:flex`)했습니다.
- **AI 큐레이션 생성 기능 및 데이터 모델 일관성 고도화 (진행 완료)**:
  - **이미지 검색 신뢰도 향상 (사진 누락 해결)**: Unsplash 검색용 키워드(`searchKeyword`) 생성 시 특정 지명뿐만 아니라 `nature, landscape, beach` 등의 시각적 풍경 키워드를 필수적으로 포함하도록 프롬프트를 고도화(`prompts.js`)했습니다. 또한 1차 검색 실패 시 영문명 기반으로 2차 재검색(Fallback)을 수행하는 방어 로직을 추가했습니다(`useLogbookAI.js`).
  - **데이터 모델 표준화 (좌표 누락 버그 해결)**: AI가 큐레이션 데이터를 생성할 때 `getCoordinatesFromAddress`를 호출하여 위/경도(`lat`, `lng`) 좌표를 획득하고 데이터를 보강하도록 개선했습니다. 이 좌표값은 DB(`saved_trips`)에 저장/업데이트 시 일반 장소와 동일하게 최상위 컬럼(`lat`, `lng`)에 적재되도록 처리하여, 큐레이션을 즐겨찾기에서 클릭했을 때 지도 좌표가 (0,0)으로 이동하던 버그를 원천 차단했습니다(`useTravelData.js`).
- **UI 개선**: 모바일 환경에서 장소 카드의 '연관 탐험지(꼬꼬무)' 기능에 접근할 수 없었던 문제를 해결하기 위해 모바일 전용 하단 플로팅 푸터 영역을 신설했습니다.
- **버그 수정 (진행 완료)**: 모바일 환경에서 새로 신설된 연관 탐험지 푸터(`fixed bottom-0`)가 부모 컨테이너의 CSS 제약(Transform 및 Backdrop-blur에 의한 Stacking Context 생성)으로 인해 화면 하단이 아닌 상단 헤더 영역으로 올라가, 탈출구(뒤로가기 버튼)와 기타 주요 UI를 덮어버리는 문제를 해결했습니다.
  - **작업 내용**: `src/components/PlaceCard/panels/PlaceChatPanel.jsx`에서 모바일 전용 연관 푸터를 `React.createPortal`을 사용하여 `document.body`로 렌더링하도록 분리했습니다. 또한, 불필요한 "연관 탐색지" 텍스트 레이블을 제거하고 버튼만 나열하도록 다이어트하여 시각적 직관성을 높였습니다.
- **버그 수정**: 큐레이션(AI 추천)을 통해 저장된 여행지가 즐겨찾기 목록(로고 패널)에서 클릭될 때, 장소 카드가 정상적으로 열리지 않는 문제를 해결했습니다.
  - **해결책**: `savedTrips`에서 항목을 매칭할 때 영문명 및 URL 파라미터(`targetId`)와 올바르게 매칭되도록 1차 방어막(Hydration) 로직을 개선했습니다.
- **전반적인 성능 최적화 (진행 완료)**: 장시간 사용 시 앱이 느려지는 현상을 해결하기 위해 렌더링 및 자원 관리 최적화를 수행했습니다.
  - **Memoization 적용**: `HomeGlobe`, `HomeUI`, `PlaceCardExpanded`, `PlaceGalleryView` 등 주요 UI 컴포넌트에 `React.memo`를 적용하여 불필요한 리렌더링을 차단했습니다.
  - **GPU 부하 감소**: 장소 카드가 열려 있을 때 지구본의 WebGL 렌더링을 중단(`display: none`)하도록 개선하여 CPU/GPU 자원을 확보했습니다.
  - **메모리 누수 해결**: `useLogbookMedia` 훅에서 생성된 Blob URL들을 언마운트 시점에 `URL.revokeObjectURL`로 해제하도록 수정했습니다.
  - **이미지 및 로직 최적화**: 갤러리 이미지에 지연 로딩(`loading="lazy"`)을 적용하고, 검색 엔진의 정적 데이터 계산 로직을 최적화했습니다.
