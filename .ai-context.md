# gateo.kr 프로젝트 컨텍스트 (AI Context)

## 1. 프로젝트 개요
- **서비스명**: gateo.kr
- **설명**: React 기반의 모던 웹 애플리케이션으로, Supabase를 백엔드로 사용하며 3D 시각화 및 AI 기능을 통합한 서비스입니다.

## 2. 주요 기술 스택
- **Frontend**: React 19.2.0, Vite, React Router DOM
- **Styling**: Tailwind CSS, Lucide React
- **Backend & BaaS**: Supabase (`@supabase/supabase-js`)
- **3D & Data Visualization**: `react-globe.gl`, `three`, `@react-three/fiber`, `@react-three/drei`
- **AI Integration**: `openai`, `@google/generative-ai`
- **State Management**: React Context API
- **Deployment**: Vercel

## 3. 폴더 구조 및 아키텍처
- `src/components/`: 재사용 가능한 UI 컴포넌트
- `src/pages/`: 라우팅되는 페이지 단위 컴포넌트
- `src/context/`: Context API를 활용한 전역 상태 관리 로직
- `src/shared/`: 공통 유틸리티, 커스텀 훅, 상수 등
- `supabase/`: Supabase 관련 설정, 데이터베이스 스키마 및 마이그레이션

## 4. 🤖 AI 어시스턴트 핵심 행동 원칙 (Core Rules)
사용자의 요청에 따라 코드를 수정하거나 기능을 추가할 때 다음 프로세스를 **엄격하게 준수**합니다:
1. **논리적 구조 제안**: 수정 및 구현하려는 코드의 논리적 구조와 방향성을 먼저 설명합니다.
2. **사용자 승인**: 제안한 구조에 대해 사용자의 승인을 대기합니다.
3. **직접 파일 수정(Edit Tool 사용)**: 승인이 완료되면 채팅창에 전체 코드를 길게 출력하며 토큰을 낭비하는 대신, AI가 에디터의 파일을 직접 수정(Edit)하여 신속하게 변경사항을 반영합니다.
4. **디자인 임의 변경 금지**: 모바일, 태블릿 대응 등 반응형 디자인 최적화를 포함하여 기존 UI/UX 디자인을 사용자의 명시적인 요청이나 합의 없이 일방적으로 변경하지 않습니다. 사용자가 직접 적용해보며 수정하는 경우가 많으므로 문제없는 디자인도 임의로 변경해서는 안 됩니다.
5. **백엔드 로직 수정 안내 의무**: Supabase Edge Function이나 DB 스키마 등 백엔드 영역의 코드를 수정한 경우, AI는 사용자에게 **"반드시 Supabase 클라우드에 업데이트(배포)해야 반영됩니다"**라는 사실과 배포 명령어(`npx supabase functions deploy <function_name>`)를 친절히 안내해야 합니다.

## 5. 최근 수정 사항 및 남은 과제

### 진행 완료된 작업
- **장소 카드 모바일/반응형 UI 최적화 (진행 완료)**:
  - **헤더 1층/2층 반응형 재구성 (`PlaceChatPanel.jsx`)**: 가로 공간이 극히 좁은 모바일에서는 1층 가로 배치(`flex-row`)로 변경하여 세로 공간을 확보하고, 태블릿 환경에서는 2층(`flex-col`), 16인치 이상 대화면에서는 다시 1층 가로 배치로 동작하도록 반응형 레이아웃을 세밀하게 재조정했습니다. 모바일에서의 텍스트 잘림 현상을 막기 위해 텍스트 `truncate` 처리 및 버튼 크기를 미세 조정했습니다.
  - **모바일 스크롤 및 겹침 현상 해결 (`PlaceGalleryView.jsx`, `PlaceWikiDetailsView.jsx`)**: 모바일에서 상단 고정 헤더와 하단 고정 푸터(연관 여행지)가 컨텐츠를 가리는 문제를 해결하기 위해, 갤러리와 백과 컨테이너에 각각 상단 패딩(`pt-24`)과 하단 패딩(`pb-28~32`)을 넉넉히 주어 컨텐츠 영역을 완전히 분리했습니다.
  - **백과(Wiki) AI 버튼 모바일 접근성 개선 (`PlaceWikiDetailsView.jsx`)**: 기존에 최하단까지 스크롤해야만 볼 수 있었던 "제미나이에게 묻기" 버튼을 모바일 환경에서는 하단 고정 푸터(`fixed bottom-0`)로 분리하여 언제든 즉시 호출할 수 있도록 변경했습니다.
  - **유튜브 영상 추천 버튼 조건부 렌더링 (`YouTubePlayerView.jsx`)**: 모바일 영상 탭에서 동영상이 정상적으로 검색되었음에도 항상 노출되던 구글폼(영상 추천) 유도 버튼을, 동영상이 없을 때(`isEmpty`)만 나타나도록 렌더링 조건을 수정했습니다.
  - **영상 하이라이트(Moments) 텍스트 잘림 해결 (`VideoInfoView.jsx`)**: 영상 탭 좌측의 모먼트 설명이 두 줄 이상일 때 잘리던 문제(`truncate`, `max-w-150px`)를 해결하여, 긴 설명글도 온전히 줄바꿈(wrap)되어 읽을 수 있도록 레이아웃을 유연하게 개선했습니다. (유튜브 검색 API 자체의 `snippet.description` 텍스트 잘림 현상 제외)
- **UI 반응형 개선 (진행 완료)**: 장소 카드의 좌측 패널 헤더 디자인을 17인치 화면 기준으로 분기 처리하여 사용성을 크게 개선했습니다.
  - **헤더 2단 분리 (`PlaceChatPanel.jsx`)**: 모바일 및 16인치 이하 태블릿 환경(`xl` 브레이크포인트 미만)에서는 좁은 가로 공간을 효율적으로 사용하기 위해 상단에 '홈, 국가명, 여행지명'을 배치하고, 하단에 '펜툴(Log), 백과, 영상/갤러리' 버튼을 가로 중앙 정렬로 2층 배치했습니다. 17인치(`xl` 이상) 모니터에서는 기존대로 넓은 1단 가로 배치가 유지됩니다.
  - **`PLACE_OVERVIEW` 제거 (`GalleryInfoView.jsx`)**: 좁은 화면에서의 세로 공간 활용도를 높이기 위해, 사진 모드(`PHOTO_LOG`)가 아닐 때 표시되던 `PLACE_OVERVIEW` 구분선 영역을 17인치 미만 환경에서 숨김 처리(`hidden xl:flex`)했습니다.
- **AI 큐레이션 생성 기능 및 데이터 모델 일관성 고도화 (진행 완료)**:
  - **이미지 검색 신뢰도 향상 (사진 누락 해결)**: Unsplash 검색용 키워드(`searchKeyword`) 생성 시 특정 지명뿐만 아니라 `nature, landscape, beach` 등의 시각적 풍경 키워드를 필수적으로 포함하도록 프롬프트를 고도화(`prompts.js`)했습니다. 또한 1차 검색 실패 시 영문명 기반으로 2차 재검색(Fallback)을 수행하는 방어 로직을 추가했습니다(`useLogbookAI.js`).
  - **데이터 모델 표준화 (좌표 누락 버그 해결)**: AI가 큐레이션 데이터를 생성할 때 `getCoordinatesFromAddress`를 호출하여 위/경도(`lat`, `lng`) 좌표를 획득하고 데이터를 보강하도록 개선했습니다. 이 좌표값은 DB(`saved_trips`)에 저장/업데이트 시 일반 장소와 동일하게 최상위 컬럼(`lat`, `lng`)에 적재되도록 처리하여, 큐레이션을 즐겨찾기에서 클릭했을 때 지도 좌표가 (0,0)으로 이동하던 버그를 원천 차단했습니다(`useTravelData.js`).
- **URL 매칭 견고성 향상 (진행 완료)**: 영문명 기반 URL 파라미터 매칭 시, 띄어쓰기와 특수문자(다국어 액센트 등)로 인해 발생하는 매칭 오류를 방지하기 위해 정규화 로직을 고도화했습니다.
  - **작업 내용**: `formatUrlName` 유틸리티 함수에 유니코드 NFD 정규화 및 발음 구별 기호(액센트) 제거 로직을 추가하고, `index.jsx`에서 `decodeURIComponent` 및 소문자 변환(lowerCase)을 적용하여 견고하고 예외 없는 라우팅 동기화를 구현했습니다.
- **UI 개선**: 모바일 환경에서 장소 카드의 '연관 탐험지(꼬꼬무)' 기능에 접근할 수 없었던 문제를 해결하기 위해 모바일 전용 하단 플로팅 푸터 영역을 신설했습니다.
- **버그 수정 (진행 완료)**: 모바일 환경에서 새로 신설된 연관 탐험지 푸터(`fixed bottom-0`)가 부모 컨테이너의 CSS 제약(Transform 및 Backdrop-blur에 의한 Stacking Context 생성)으로 인해 화면 하단이 아닌 상단 헤더 영역으로 올라가, 탈출구(뒤로가기 버튼)와 기타 주요 UI를 덮어버리는 문제를 해결했습니다.
  - **작업 내용**: `src/components/PlaceCard/panels/PlaceChatPanel.jsx`에서 모바일 전용 연관 푸터를 `React.createPortal`을 사용하여 `document.body`로 렌더링하도록 분리했습니다. 또한, 불필요한 "연관 탐색지" 텍스트 레이블을 제거하고 버튼만 나열하도록 다이어트하여 시각적 직관성을 높였습니다.
- **버그 수정**: 큐레이션(AI 추천)을 통해 저장된 여행지가 즐겨찾기 목록(로고 패널)에서 클릭될 때, 장소 카드가 정상적으로 열리지 않는 문제를 해결했습니다.
  - **해결책**: `savedTrips`에서 항목을 매칭할 때 영문명 및 URL 파라미터(`targetId`)와 올바르게 매칭되도록 1차 방어막(Hydration) 로직을 개선했습니다.
- **전반적인 성능 최적화 (진행 완료)**: 장시간 사용 시 앱이 느려지는 현상을 해결하기 위해 렌더링 및 자원 관리 최적화를 수행했습니다.
  - **Memoization 적용**: `HomeGlobe`, `HomeUI`, `PlaceCardExpanded`, `PlaceGalleryView` 등 주요 UI 컴포넌트에 `React.memo`를 적용하여 불필요한 리렌더링을 차단했습니다.
  - **GPU 부하 감소**: 장소 카드가 열려 있을 때 지구본의 WebGL 렌더링을 중단(`display: none`)하도록 개선하여 CPU/GPU 자원을 확보했습니다.
  - **메모리 누수 해결**: `useLogbookMedia` 훅에서 생성된 Blob URL들을 언마운트 시점에 `URL.revokeObjectURL`로 해제하도록 수정했습니다.
  - **이미지 및 로직 최적화**: 갤러리 이미지에 지연 로딩(`loading="lazy"`)을 적용하고, 검색 엔진의 정적 데이터 계산 로직을 최적화했습니다.
- **지구본 시각화 및 데이터 구조 개선 (진행 완료)**:
  - **테마 기본값 수정**: 초기 렌더링 시 지구본 테마가 '네온(neon)'으로 노출되던 문제를 '딥모드(deep)'가 기본으로 적용되도록 상태 및 컴포넌트 초기값을 수정했습니다(`index.jsx`, `HomeGlobe.jsx`).
  - **여행지 데이터 전 지구적 안배**: 한 지역(특히 아시아나 유럽)에 마커가 밀집되어 지구본 탐색의 재미가 떨어지던 현상을 해결하기 위해, 모든 카테고리의 데이터를 6개 대륙 전역으로 고르게 재분배했습니다(`travelSpots.js`).
  - **'Nearby' 카테고리 개편**: 아시아 지역 편중의 주원인이었던 '근거리(Nearby)' 카테고리를 전 세계 랜드마크를 아우르는 '문화/역사(Culture)' 카테고리로 변경하여 다채로운 탐색 환경을 마련했습니다(`HomeUI.jsx`, `index.jsx`).
  - **영상 데이터 매칭 로직 고도화**: 장소 데이터(`travelSpots.js`) 수정 시 고유 숫자 ID가 변경되어 영상 데이터(`travelVideos.js`)와의 연결이 끊어지는 취약점을 개선하기 위해, `travelVideos.js`의 객체 Key를 숫자 ID에서 한글 장소명(`name`)으로 전면 치환했습니다. 기존 검색 훅(`useYouTubeSearch.js`)은 이미 이름 기반 탐색을 지원하므로 확장성과 유지보수성이 크게 향상되었습니다.
- **보안 강화 및 API 통신 구조 개선 (진행 완료)**:
  - **YouTube API 호출 서버 위임**: 클라이언트 단(`src/shared/api/youtubeClient.js`)에 노출되어 있던 `YOUTUBE_API_KEY` 환경변수를 제거하고, 검색 로직과 DB Upsert 과정을 새로운 Supabase Edge Function(`fetch-place-videos`)으로 분리(위임)하여 완벽한 보안을 확보했습니다.
  - **Supabase 인증(RLS) 우회 설정**: 누구나 접근 가능한 장소 백과(`update-place-wiki`)와 영상 검색(`fetch-place-videos`) Edge Function이 로그인하지 않은 사용자에게 401 Unauthorized 에러를 발생시키는 문제를 해결하기 위해 `supabase/config.toml`에 `verify_jwt = false` 설정을 추가했습니다.
  - **데이터베이스 RLS 정책 확립**: 클라이언트에서 직접 접근하는 개인 데이터 테이블(`reports`, `saved_trips`, `memos`)의 무단 조작을 막고, 서버 전용 테이블(`place_videos`, `place_wiki`)을 외부 공격으로부터 보호하기 위한 최적의 Row Level Security(RLS) SQL 정책을 점검 및 확립했습니다.

- **장소 데이터 정규화 및 검색 정확도 향상 (진행 완료)**:
  - `travelSpots.js`와 `citiesData.js` 간의 장소명 및 국가명 데이터 불일치 해결 완료.
  - `citiesData.js` 내 누락된 `country` 및 `country_en` 필드 일괄 추가, `name_en`에 포함된 불필요한 꼬리표 분리.
  - `travelSpots.js` 내 다중 국가 표기 등 검색 API에 혼동을 주는 부분을 단일 국가로 통일.
  - 지오코딩 및 API 호출 시 (Unsplash 사진, YouTube 영상, Gemini 장소 백과) 정규화된 `name` + `country`를 조합하여 엉뚱한 장소가 호출되는 오류 원천 차단.

- **지구본 렌더링 성능 최적화 및 데이터 병합 (Data & Performance) (진행 완료)**:
  - **데이터 승격 및 확장**: 5개의 카테고리(휴양, 자연, 도시, 문화, 모험)마다 전 세계 6대륙에서 1개씩 추가로 발탁하여, 각 섹터당 12개, 총 60개의 아이콘 마커(`travelSpots.js`)로 확장 배치했습니다.
  - **데이터 겹침 원천 차단**: 아이콘 마커로 승격된 60개의 여행지를 텍스트 라벨 풀(`citiesData.js`)에서 완전히 삭제하여, 텍스트 라벨과 아이콘이 겹쳐 보이는 시각적 오류와 데이터 중복을 제거했습니다.
  - **DOM 부하 해결 (True LOD 도입)**: 줌인 시 HTML 마커를 투명도(`opacity: 0`)로만 숨기던 기존 방식을 개선하여, `HomeGlobe.jsx`에서 줌인 고도(`lodLevel === 1`) 진입 시 `htmlElementsData`를 빈 배열(`[]`)로 넘기도록 수정했습니다. 이로 인해 수십 개의 DOM 노드가 완전히 언마운트되어 프레임 드랍 현상이 극적으로 해소되었습니다.
- **지구본 오클릭 및 튕김 현상 방지 (UX/Interaction) (진행 완료)**:
  - **스마트 스냅(Smart Snap) 기능 도입**: 태평양 등 지명이 없는 오지나 바다를 클릭했을 때 "미지의 탐험지"라는 불필요한 카드가 열리는 대신, 클릭한 지점에서 가장 가까운 육지나 도시로 자석처럼 끌어당겨(Snap) 의미 있는 장소 카드가 자동으로 열리도록 로직을 전면 개편했습니다(`useHomeHandlers.js`).
  - **스무스 로딩 제스처 및 피드백 적용**: 지구본 클릭 시 Geocoding API 응답을 기다리는 동안의 시각적 공백을 메우기 위해 클릭 즉시 물방울(Ripple) 애니메이션을 발생시키고, 장소 카드에 "위치 탐색 중..."이라는 뼈대(Skeleton) UI를 선제적으로 띄워 다중 클릭 공격과 앱 마비 오해를 원천 차단했습니다.
  - **모바일 무한 로딩 및 마비 버그 해결**: 
    - 로딩 중 나타나는 유령 핀(위치 탐색 중...)이 지구본 배열에 남아 모바일 화면에서 이중 클릭되거나 무한 로딩을 유발하던 버그를, `useGlobeLogic.js`의 `addScoutPin` 함수 내 가비지 컬렉터(Garbage Collection) 로직을 통해 새 핀 삽입 시 자동 소멸되도록 완벽히 해결했습니다.
    - 모바일 등 저사양 환경에서 다중 클릭 시 물방울 효과(Ripple)와 카메라 비행 애니메이션이 중첩되며 WebGL 컨텍스트가 멈추고 터치가 마비되는 현상을 해결하기 위해, 클릭 즉시 발생하는 시각적 피드백을 가벼운 1회성 Ripple로 제한하고 불필요한 `moveToLocation` 호출을 제거(디바운싱)하여 성능 부하를 없앴습니다.

- **스마트 검색창 AI 자동 교정 엔진 및 AI 모델 최적화 (진행 완료)**:
  - **Smart Search Fallback 도입**: 사용자가 오타나 미등록 지명을 검색하여 Geocoding이 실패할 경우, 막히는 대신 백그라운드 AI가 개입하여 "정확한 지명 유추 및 교정"을 거친 뒤 올바른 장소로 조용히(Silent) 연결시켜주는 직관적 시스템을 `useHomeHandlers.js`에 구축했습니다.
  - **AI 모델 용도별 세분화(Model Routing)**: 앱 내 AI 호출 구간을 성격에 맞게 분리하고 최적의 모델을 배정했습니다.
    - `스마트 검색 교정`: 가장 빠르고 비용이 적게 드는 **`gemini-1.5-flash-8b`** (초고속 단답형)
    - `장소카드 AI 채팅`: 대화의 문맥 유지와 자연스러운 반응이 중요한 롤플레잉용 **`gemini-2.5-flash`**
    - `장소 백과 실전 팁`: 최신 정보와 높은 신뢰도가 요구되는 Edge Function은 **`gemini-1.5-pro`** (최상위 안정성)
    - `리포트 페이지 블로그 작성`: 수준 높은 추론과 감성적인 글쓰기 스킬이 필요한 작업은 **`gemini-2.5-pro`**
    - `리포트 페이지 큐레이션`: 정형화된 JSON 데이터를 일관되게 반환하는 작업은 **`gemini-2.0-flash`** (밸런스형)
  - **백과(Wiki) AI 실용 정보 제공 기능(UX/Prompt) 개선**:
    - **가짜 로딩(Fake Loading) 연출**: 캐시된 실용 정보가 즉시 나타나며 발생하는 '전문성 결여' 느낌을 없애기 위해, `PlaceWikiDetailsView.jsx`에 1.5초간의 로딩 스피너를 강제 연출하여 AI가 실시간으로 분석하는 듯한 감성을 추가했습니다.
    - **실용 정보 프롬프트 개편**: 기존 매거진 스타일의 서정적인 백과 내용과 대비되도록, "여기는 도대체 어떤 곳인가요?"에 대한 1분 요약(정체성과 핵심 매력)을 프롬프트 최상단에 강제하여 여행자의 근본적인 궁금증을 속 시원히 해결하도록 개선했습니다.
