# gateo.kr 프로젝트 컨텍스트 (AI Context)

## 1. 프로젝트 개요
- **서비스명**: gateo.kr
- **설명**: React 기반의 모던 웹 애플리케이션으로, Supabase를 백엔드로 사용하며 3D 시각화 및 AI 기능을 통합한 서비스입니다.

## 2. 주요 기술 스택
- **Frontend**: React 19.2.0, Vite, React Router DOM
- **Styling**: Tailwind CSS, Lucide React
- **Backend & BaaS**: Supabase (`@supabase/supabase-js`)
- **3D & Data Visualization**: `react-globe.gl`, `three`, `@react-three/fiber`, `@react-three/drei`
- **AI Integration**: `openai`, `@google/generative-ai`
  - **사용 가능한 Gemini 모델 목록 (2026.03.12 기준)**:
    - `gemini-3.1-pro` (텍스트 출력 모델)
    - `gemini-2.5-pro` (텍스트 출력 모델)
    - `gemini-2.5-flash` (텍스트 출력 모델)
    - `gemini-2.5-flash-lite` (텍스트 출력 모델)
    - `gemini-3.1-flash-lite` (텍스트 출력 모델)
    - 기타 모델들은 API 콘솔 참조
- **State Management**: React Context API
- **Deployment**: Vercel

## 3. 폴더 구조 및 아키텍처
- `src/components/`: 재사용 가능한 UI 컴포넌트
- `src/pages/`: 라우팅되는 페이지 단위 컴포넌트
- `src/context/`: Context API를 활용한 전역 상태 관리 로직
- `src/shared/`: 공통 유틸리티, 커스텀 훅, 상수 등
- `supabase/`: Supabase 관련 설정, 데이터베이스 스키마 및 마이그레이션

## 4. 🤖 AI 어시스턴트 핵심 행동 원칙 (Core Rules)
사용자의 요청에 따라 코드를 수정하거나 기능을 추가할 때 다음 프로세스를 **엄격하게 준수**합니다:
1. **논리적 구조 제안**: 수정 및 구현하려는 코드의 논리적 구조와 방향성을 먼저 설명합니다.
2. **사용자 승인**: 제안한 구조에 대해 사용자의 승인을 대기합니다.
3. **직접 파일 수정(Edit Tool 사용)**: 승인이 완료되면 채팅창에 전체 코드를 길게 출력하며 토큰을 낭비하는 대신, AI가 에디터의 파일을 직접 수정(Edit)하여 신속하게 변경사항을 반영합니다.
4. **디자인 임의 변경 금지 및 UI 밀도 원칙**: 모바일, 태블릿 대응 등 반응형 디자인 최적화를 포함하여 기존 UI/UX 디자인을 사용자의 명시적인 요청이나 합의 없이 일방적으로 변경하지 않습니다.
   - **스크롤 최소화 및 중요 버튼 누락 방지 원칙**: 장소 리뷰 등 작성하기/편집창 뷰를 구성할 때 전체 스크롤을 최소화(Scroll-Free)하여, 공간 효율성을 위해 패딩/글자크기/버튼크기를 최적화하고 한 화면에 정보를 담아야 합니다. (예: 전체공개 버튼이나 등록 버튼이 스크롤 아래에 숨어 의도치 않게 확인 없이 등록되는 문제를 방지할 것).
   - **타이포그래피 및 단어 끊김 방지 원칙**: 한국어 텍스트가 렌더링되는 문단(`p`, `div` 등)에는 단어가 중간에서 무분별하게 끊겨 줄바꿈되는 현상을 방지하기 위해, 가독성을 고려하여 반드시 CSS 속성(`word-break: keep-all;` 또는 Tailwind 클래스 `break-keep`)을 명시적으로 적용해야 합니다.
5. **백엔드 로직 수정 안내 의무**: Supabase Edge Function이나 DB 스키마 등 백엔드 영역의 코드를 수정한 경우, AI는 사용자에게 **"반드시 Supabase 클라우드에 업데이트(배포)해야 반영됩니다"**라는 사실과 배포 명령어(`npx supabase functions deploy <function_name>`)를 친절히 안내해야 합니다.
6. **데이터 구조 및 Hydration 규칙 엄수**: DB에서 불러온 동적 장소 데이터(`savedTrips` 등)는 장소명이 `destination` 컬럼으로 관리되나, `PlaceCard` 등의 UI 컴포넌트는 전역 상태에서 `location.name` 속성을 필수로 요구합니다. 라우팅 로직이나 컴포넌트 상태 설정(`setSelectedLocation` 등) 시, **반드시 원본 DB 객체를 복사한 후 `name: target.destination` 등으로 필수 UI 속성을 매핑(Hydration)** 하는 과정을 거쳐야 합니다. 누락 시 장소 카드가 빈 화면으로 렌더링되는 치명적 버그가 잦게 재발하므로 데이터 흐름 변경 시마다 각별히 주의해야 합니다.
7. 🚨 **갤러리(이미지 검색/페이지네이션) 로직 수정 시 절대 주의**: 
   - 사진 갱신(Refresh) 로직이나 검색어 정제 로직을 변경할 때, **기존에 잘 나오던 사진이 갑자기 0개가 되거나 빈약해지는 문제(regression)가 수차례 반복**되었습니다. 
   - API 검색어 최적화 시 오지/자연경관(예: 에베레스트, 통가) 등은 한글 검색으로 결과가 나오지 않으므로 영문 매핑(`FALLBACK_DICTIONARY`)을 함부로 제거하지 말고 보존해야 합니다.
   - '강제 새로고침(Force Refresh)' 등 페이지네이션 로직을 짤 때는 새로운 결과를 기존 결과에 덮어씌워버리면(Replace) 사진 수가 줄어들어 빈약해 보이게 되므로, 반드시 **기존 사진들에 새 사진들을 누적(Append)** 하는 방식을 유지해야 합니다.
8. **작업 단위 분할 및 커밋 주도 원칙 (Commit-Driven Workflow)**: 작업 효율성을 위해 한 번에 여러 파일이나 여러 기능을 변경하는 것은 허용하지만, 사후 리뷰와 이력 관리가 어려워지지 않도록 **논리적 단위(하나의 기능 완성 또는 하나의 주요 파일 완성)** 로 작업 단계를 명확히 분할하여 순차적으로 진행합니다. 너무 많은 이질적인 기능들을 한 번에 무더기로 수정하고 결과를 통보하는 것을 지양합니다. 하나의 단위 작업이 완료될 때마다 사용자에게 **점검해야 할 사항(테스트 포인트)** 과 해당 변경 사항에 대한 **`git commit -m "커밋 메시지"`** 를 제안하여 사용자가 즉각적으로 버그 확인 및 커밋을 진행할 수 있도록 돕습니다.
9. **한글(영문) 고유명사 표기 규칙**: AI가 장소나 상호명 등 고유명사를 생성하거나 답변할 때는 항상 `한글명('현지 영문명')` 형식을 엄수합니다. 스마트 툴킷이나 로컬 가이드 텍스트 등에서 이 표기법을 통해 검색의 일관성과 가독성을 높입니다.

  - **마커 연결 오류 및 요약 카드 UI 수정**: 지명(Name)이 국가명(Country)보다 최우선 매칭되도록 로직 분리. 대화면 모바일 기기(아이폰 14 Pro Max 등)에서 발생하던 요약 카드 왼쪽 쏠림 현상을 화면 중앙 여백 기반 반응형으로 수정.
  - **스마트 검색 고도화 및 좌표 파싱**: AI 텍스트 반환을 JSON 포맷(위경도, 한/영 지명 및 국가)으로 변경하여 추가 좌표 검색(Geocoding) API 호출 생략 및 속도 최적화. `FALLBACK_DICTIONARY`에 누락 한글명 추가하여 갤러리 검색 오류 원천 차단.
  - **유튜브 검색 폴백 로직 고도화**: 1차 검색 시 `videoCaption: 'closedCaption'` 조건 제거. 0개일 경우를 대비해 프론트엔드에서 파악한 영문명(`name_en`)을 `fallbackQuery`로 백엔드에 넘겨 영문명 기반 `travel vlog` 2차 재검색 지원. (Edge Function 연동 완료)

- [x] **[Phase 3] 정보 갱신 아키텍처 완전 개편 (Defensor 폐기 및 Lazy Update 적용)**
  - **Defensor(Flash) 폐기 및 Pro 통합**: 정보 갱신 감지 시 퀄리티 저하를 막기 위해 Flash 모델 로직을 제거하고, 오직 `gemini-2.5-pro` 모델만을 활용하도록 Edge Function(`update-place-wiki`) 리팩토링.
  - **NO_CHANGES 프롬프트**: Pro 모델에게 기존 정보를 제공하고 중대 변동 사항이 없을 경우 `NO_CHANGES`만 반환하도록 지시하여 출력 비용 최소화.
  - **수동 버튼 제거 및 14일 주기 자동 갱신(Lazy Update)**: 사용자의 수동 갱신 피로도를 없애기 위해 '최신 정보로 묻기' 버튼을 완전히 제거. 대신 `ai_info_updated_at` 기준 14일이 경과한 장소를 누군가 **조회할 때만** 백그라운드에서 조용히 갱신이 일어나도록 최적화.
  - **텍스트 가독성 및 형식 통일**: 마크다운 헤딩(#) 사용 금지 및 불릿 포인트(*) 사용 강제. 고유명사 `한글명('영문명')` 표기 규칙 강제.
  - **JSON 파싱 안정성 및 줄바꿈/여백 개선**: 빈번한 `JSON Parse Error`를 막기 위해 이스케이프 처리를 프롬프트로 강제하고, 각 항목(`*` 불릿 포인트) 사이마다 `\n\n` 여백을 의도적으로 추가하여 가독성 대폭 향상 및 TDZ(Temporal Dead Zone) 에러 픽스.
