## 1. 유지 규약 (AI·세션 종료 시)

### 1.1 목적

- 사용자 인수인계서가 아니라 **다음 AI 세션을 위한 압축 메모리**.
- 「무엇을 했나」보다 **「지금 무엇이 맞고, 무엇을 하면 안 되나」**.

### 1.2 분량

- **목표 ~200줄, 상한 ~250줄**. 넘으면 **5절·6절**부터 잘라낸다.
- 한 항목은 **1~3문장**. 표·체크리스트·npm 명령은 6절에만 두어도 됨.

### 1.3 절별 — 넣을 것 / 넣지 말 것

| 절 | 넣을 것 | 넣지 말 것 |
|----|---------|------------|
| **0** | 사용법·자동 로드 여부 | — |
| **1** | 이 규약 | 작업 내역 |
| **2~3** | 스택·폴더·**장애 방지** 규칙 | 완료된 기능 설명 장문 |
| **4** | 협업·커밋·UI 변경 금지 등 | — |
| **5** | **현재 스냅샷** 5~10줄, **미완·리스크**만 | 완료 로그·SHA 나열·Phase 타임라인 |
| **6** | **남은 과제**·운영 SSOT·npm·체크리스트 | 완료된 Phase 상세 |
| **7** | 문서·일지 루틴 | — |

### 1.4 갱신 시점

**갱신 O**

- 아키텍처·SSOT 파일·우선순위가 바뀜 (예: slug-first, Mapbox PROD 고정).
- **미완**·**배포 대기**·**재배포 필요**가 생기거나 해소됨.
- 새 에이전트가 **같은 실수를 반복**할 만한 규칙이 생김 (3절에 추가).

**갱신 X** (일지만)

- 버그 수정·UI 미세 조정·단일 커밋 완료.
- 「완료·배포됨·QA 통과」만 있는 항목 → 5절에서 **삭제**하고 일지에만 남김.

### 1.5 세션 종료 체크리스트 (AI)

1. `git status` — 사용자가 commit 요청 시에만 **한글** 메시지로 commit.
2. `plans/YYYY-MM-DD-project-log.md` — **오늘 2~5줄** (없으면 새 파일, 직전 일지 링크).
3. **릴리스 노트** — 아래 **1.7** (사용자-facing 변경이 있을 때).
4. `.ai-context.md` — **필요할 때만**: 상단 날짜, **5절 3줄 이내**, **6절** 미완/완료 1줄 조정.
5. **새 `plans/*-완료보고.md` 생성 금지** — 사용자가 명시 요청한 경우만.

### 1.6 금지

- 5절에 **이력 누적** (파일 자체가 「쌓지 말 것」을 규정함).
- 일지·handoff 문서 **전체 복사**.
- 사용자 **테스트·승인 전** 「완료」 단정 (4절).
- 릴리스 노트 **사용자 합의 없이** [`releaseNotes.js`](src/data/releaseNotes.js) 반영.

### 1.7 릴리스 노트 (세션 종료·배포 전)

**SSOT**: [`src/data/releaseNotes.js`](src/data/releaseNotes.js) — `SiteUpdateBanner`(홈 팝업) · FooterModal **Updates** 탭.

**대상**: 사용자-facing 변경(새 기능·제휴 입점·UI 개선 등). 내부 리팩터·버그만·사소한 조정은 **생략 가능**.

**절차** (4절 완료 단정과 동일 — **합의 후 반영**):

1. 세션 종료(또는 배포 직전) AI가 **초안을 대화에 먼저 제시** — `id`, `category`, `title`, `items`.
2. 사용자 **승인·수정 합의** 후에만 파일에 추가 (`RELEASE_NOTES` **배열 맨 앞**).
3. `id`(보통 `YYYY-MM-DD`)가 바뀌면 「다시 보지 않기」 사용자에게도 팝업 재표시.

**category**: `feature` · `partner` · `fix` · `notice` (선택).

## 4. AI 협업 원칙

1. **기획 .md 남발 금지** — 사용자 명시 없으면 `plans/`에 새 완료보고/대형 계획서 만들지 않음. 일지 또는 본 파일 5~6절만.
2. **구조 합의 → 워크스페이스 직접 편집** — 채팅에 수백 줄 덤프 지양.
3. **UI 임의 변경 금지** — 레이아웃·반응형·모바일 승인 없이 변경 안 함. 한국어 `break-keep` 등 기존 규칙 유지.
4. **완료 단정** — 사용자 테스트·승인 없으면 「제안/대기」. commit은 **제안**만 (요청 시 실행).
5. **고유명사**: `한글명('English name')`.
6. **도메인 로직** (툴킷·갤러리·제휴): 3절·6절·해당 소스·Edge Function 우선.
7. **Git 커밋 메시지**: **한글** (Conventional prefix는 선택).

**런타임 우선순위**: 수동 오버라이드 → live `essential_guide` → toolkit-sync JSON → runtime-infer → multi-rule·좌표.

```powershell
cd c:\dev\days
npm run sync:airports-from-toolkit   # DB 읽기만, 툴킷 반영
npm run generate:airports            # spots 재생성 + TRAVEL_SPOT_PLACE_ID_OVERRIDES → placeIds 병합
npm run generate:flights             # flight-booking-overrides → travelSpotFlightBookings.json
npm run audit:airports
npm run audit:flight-route-gaps   # Phase 0+ gap-report (271 slug routeKind + uiPlace + airportsIndex)
npm run generate:graph-direct-review   # graph-direct 78건 검수 MD (gap-report 선행)
npm run import:airports           # OurAirports → Supabase airports
npm run import:routes             # OpenFlights → Supabase air_routes (migration 선행)
npm run generate:airports-index   # airportsIndex.json (scheduled, rental 제외)
npm run generate:flight-routes    # OpenFlights graph → travelSpotFlightRoutes.json
npm run audit:flight-routes       # graph vs corridor/override semantic audit
npm run audit:flight-route-detours   # detour ratio · cross-track (graph-only 검수)
npm run db:apply-migrations       # migration SQL (SUPABASE_DB_PASSWORD; IPv6 불가 시 pooler 자동 폴백)
```

**`generate:airports` 우선순위**: overrides.mjs → JSON `curated-override`(overrides에 slug 없을 때) → `toolkit-sync` → infer. **`spots` 직접 수정 금지** — overrides 후 generate. **배너 원복 방지**: JSON·`RENTAL_MULTI`만 고치지 말 것 — `travel-spot-airport-overrides.mjs`에 slug 등록(다관문은 `searchHintIatas`+`bannerNote`). **`tripFlightArrivalIata`**: Trip 도착만 분리 · `preferredLinkIata` = 렌터카·픽업 최종.
