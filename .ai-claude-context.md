# Claude(Roo) 전용 컨텍스트 설정

**대상**: Claude Sonnet 4.5 (VSCode Roo Extension)
**목적**: Claude 특화 워크플로우 최적화 및 토큰 관리
**최종 업데이트**: 2026-03-31

---

## 🚨 핵심 규칙: 다음 세션 제시어 제공

**모든 작업 종료 시 (attempt_completion) 반드시 포함:**

```
📌 다음 세션 시작 제시어:
"[참조 파일명] 확인하고 [다음 작업]"
```

**예시:**
```
📌 다음 세션 시작 제시어:
".ai-claude-context.md와 phase9-2-MASTER-GUIDE.md 확인하고 Phase 2 실행"
```

또는

```
📌 다음 세션 시작 제시어:
".ai-claude-context.md 확인 후 여행지 100개 추가 작업 시작"
```

**이유**: VSCode Roo는 컨텍스트 파일을 자동 로드하지 않으므로, 사용자가 명확한 지시어로 시작해야 토큰 낭비를 방지할 수 있습니다.

## 🎯 Claude 모드별 역할 정의

### 🏗️ Architect 모드
**목적**: 정보 수집, 분석, 계획 수립  
**출력물**: TODO 리스트 (주) + 짧은 문서 (부)

**핵심 원칙**:
1. **TODO 리스트 우선 전략**
   - 가능하면 `update_todo_list` 도구 사용
   - 체크리스트 형태로 작업 항목 관리
   - 진행 상황 실시간 업데이트

2. **문서 작성 최소화**
   - 기본 목표: 50줄 이하
   - 허용 한도: 200줄 이하
   - 예외: 프롬프트 모음, 기술 분석 문서

3. **중복 방지 규칙**
   - 같은 내용을 여러 파일에 작성 금지
   - 신규 생성보다 기존 문서 업데이트 우선
   - 코드 전체 복사 금지 (파일 경로:라인 번호로 참조)

4. **파일명 규칙** ⭐ 신규
   ```
   ✅ 날짜 기반 정렬 (시간순 관리):
   - 2026-03-31-project-log.md          (진행 로그)
   - 2026-03-31-globe-overlap.md        (세션 기록)
   - 2026-03-30-phase9-completion.md    (완료 보고)
   
   ✅ 참고용 계획 (날짜 불필요):
   - phase9-2-ai-prompts.md             (재사용 프롬프트)
   - globe-optimization-strategy.md     (전략 문서)
   
   ❌ 금지:
   - phase9-2-comprehensive-final-v3-MASTER-GUIDE.md
   - next-session-guide.md
   - completion-report.md
   ```

5. **문서 유형별 관리 전략** ⭐ 신규
   
   **A. 누적 진행 로그 (1개 파일 지속 업데이트)**
   ```
   plans/2026-03-31-project-log.md
   
   ✅ 매일 세션마다 업데이트
   ✅ 완료 작업 추가 기록
   ✅ 다음 작업 TODO 업데이트
   ❌ 신규 completion 파일 생성 금지
   ```
   
   **B. 참고용 계획 (신규 생성 허용)**
   ```
   phase9-2-ai-prompts.md              (AI 프롬프트 모음)
   globe-optimization-strategy.md      (기술 분석/전략)
   
   ✅ 재사용 가능한 내용
   ✅ 참고 자료로 계속 활용
   ❌ 일회성 내용 작성 금지
   ```
   
   **C. 세션 기록 (날짜 기반, 간결하게)**
   ```
   2026-03-31-globe-overlap-fix.md     (최대 100줄)
   
   ✅ 특정 세션의 주요 결정사항
   ✅ 문제 해결 과정 기록
   ❌ 상세 코드 복사 금지
   ❌ 장황한 설명 금지
   ```

6. **문서 구조 템플릿**
   ```markdown
   # [작업명] - [날짜]

   ## 목표
   - 한 줄 요약

   ## 작업 항목
   - [ ] 작업 1
   - [ ] 작업 2

   ## 참고
   - path/to/file.js:10-20
   ```

### 💻 Code 모드
**목적**: 실제 코드 구현 및 수정  
**입력**: TODO 리스트 또는 사용자 직접 지시

**독립성 인지**:
- Architect 계획서는 참고용 (강제력 없음)
- 필요한 파일을 독자적으로 읽고 판단
- 사용자 지시를 최우선으로 따름

## 📊 토큰 관리 전략

### 현재 제약사항
- **분당 리미트**: 30,000 토큰
- **컨텍스트 윈도우**: 200,000 토큰

### 토큰 절약 기법

#### 1. 파일 읽기 최적화
```javascript
// ❌ 전체 파일 읽기
read_file({ path: 'large-file.js' })

// ✅ 필요한 부분만
read_file({ 
  path: 'large-file.js',
  mode: 'indentation',
  indentation: { anchor_line: 42 }
})
```

#### 2. 검색 최적화
```javascript
// ❌ 광범위 검색
search_files({ path: '.', regex: '.*', file_pattern: '*' })

// ✅ 타겟팅 검색
search_files({ 
  path: 'src/components', 
  regex: 'function handleClick',
  file_pattern: '*.jsx'
})
```

#### 3. 문서 읽기 제한
- 한 번에 최대 3개 계획 문서만 읽기
- 500줄 이상 문서는 필요한 섹션만 읽기
- 중복 내용 문서는 하나만 선택

#### 4. 출력 압축
```markdown
❌ 장황한 설명:
"이 함수는 사용자의 입력을 받아서 데이터베이스에 저장하는 기능을 수행합니다. 
먼저 입력값을 검증하고, 그 다음에 데이터를 변환하여..."

✅ 간결한 설명:
"입력 검증 → DB 저장 함수"
```

## 📁 문서 관리 규칙

### 생명주기 관리

```
신규 작성 → 누적 업데이트 → 완료 후 아카이브
```

#### 폴더 구조
```
plans/
├── YYYY-MM-DD-project-log.md      # 누적 진행 로그 (지속 업데이트) ⭐
├── YYYY-MM-DD-session-*.md        # 세션 기록 (간결)
├── phase*-prompts.md              # 참고용 재사용 문서
├── *-strategy.md                  # 기술 분석/전략
└── archive/                       # 완료 작업 아카이브
```

#### 파일 관리 원칙 ⭐ 개선
- **누적 로그 업데이트**: 진행사항은 `project-log.md`에 계속 추가
- **세션 기록**: 중요 결정사항만 날짜 기반 파일로 간결히 작성
- **참고 문서**: 재사용 가능한 프롬프트, 전략만 신규 생성
- **완료 문서**: 즉시 archive/ 이동
- **불필요 문서**: completion, summary, report 등 생성 금지

### Phase 작업 문서 패턴

#### ❌ 기존 (중복 과다)
```
phase9.md
phase9-1-plan.md
phase9-1-guide.md
phase9-1-report.md
phase9-1-summary.md
phase9-2-plan.md
phase9-2-guide.md
phase9-2-comprehensive.md
phase9-2-MASTER-GUIDE.md
...
```

#### ✅ 개선 (누적 + 참고)
```
2026-03-31-project-log.md       # 모든 진행사항 누적 ⭐
phase9-2-ai-prompts.md          # AI 프롬프트 (재사용)
phase9-2-MASTER-GUIDE.md        # 통합 가이드 (참고용)

❌ 생성 금지:
- phase9-completion-report.md
- phase9-session-summary.md
- phase9-next-session-guide.md
```

## 🚫 금지 패턴

### 1. 일회성 보고서/요약 파일 생성 ⭐ 강화
```
❌ 신규 생성 금지:
- *-completion-report.md
- *-session-summary.md
- *-next-session-guide.md
- *-implementation-guide.md

✅ 대신:
- 2026-03-31-project-log.md에 누적 업데이트
```

### 2. MASTER/COMPREHENSIVE 남발
```
❌ phase9-MASTER-GUIDE.md
❌ phase9-comprehensive-action-plan.md
❌ phase9-final-ultimate-guide-v3.md

✅ 기존 문서 업데이트 또는 project-log.md 사용
```

### 3. BEFORE/AFTER 코드 전체 복사
```markdown
❌ 400줄 코드 블록:
// BEFORE
[전체 코드 200줄]

// AFTER  
[전체 코드 200줄]

✅ 참조 링크:
수정: `src/App.jsx:42-45`
변경 내용: filter 추가
```

### 3. 여러 문서에 같은 내용 반복
```
❌ guide.md, plan.md, summary.md에 동일한 테스트 항목

✅ 하나의 문서에만 작성
```

### 4. 예상 시간 추정
```
❌ "이 작업은 4-6시간이 소요됩니다"
✅ 시간 추정 없이 작업 항목만 나열
```
