# 2026-04-14 프로젝트 진행 로그

## 1. 진행 내용
### [Phase 8-4] 클룩(Klook) 다이렉트 어필리에이트 동적 딥링크 연동 완료
- **작업 목적**:
  - 기존 Travelpayouts 클룩 링크 우회 시 발생하는 파라미터 유실 및 방화벽 차단 문제를 해결하고, 장소카드에서 장소명을 바탕으로 목적지 맞춤 검색 결과를 제공하기 위함.
- **주요 구현 사항**:
  - 클룩 공식 파트너 검색 위젯(iframe) 도입을 시도했으나 브라우저 보안 제약(Same-Origin Policy, iframeResizer 에러 등) 및 외부 검색어 자동 기입(data-keyword 등)을 공식 지원하지 않는 구조적 한계 확인.
  - 이에 따라 위젯을 롤백하고, Travelpayouts를 경유하지 않는 **클룩 다이렉트 어필리에이트 URL (aff_adid=1256120)**을 활용한 동적 딥링크 방식으로 선회.
  - `src/components/PlaceCard/tabs/PlannerTab.jsx`의 `getMultiLinks` 로직 업데이트:
    - 교통/패스(`transport`): 해당 여행지명으로 클룩 전체 검색 페이지 맵핑
    - 유심/와이파이(`connectivity`): "여행지명 + 유심"으로 클룩 검색 페이지 맵핑
    - 페리 예약(`ferry_booking`): "여행지명 + 페리"로 클룩 검색 페이지 맵핑
- **기대 효과**:
  - 장소별 클릭 이벤트가 발생할 때 클룩 검색어에 장소명이 자동으로 주입되어 렌더링되므로, 사용자의 이탈률 감소 및 전환율 상승 기대.
  - 클룩 고유 어필리에이트 캠페인(`1256120`)의 직접 트래킹으로 데이터 유실 방지 및 통계 정확도 강화.

### [Phase 8-4] 모빌리티(공항 픽업/렌터카) 제휴 위젯 롤백 및 동적 딥링크 개편
- **작업 목적**: 
  - 공항 픽업 위젯(Intui.travel, GetTransfer) 도입을 시도했으나, 제공사 스크립트의 외부 파라미터(장소명) 기입 및 한국어 강제 적용 미지원 한계를 확인하여 UX를 개선하기 위함.
- **주요 구현 사항**:
  - `IntuiWidget` 및 `GetTransferWidget` 컴포넌트 전체 삭제.
  - 길리메노 등 직항 공항이 없는 섬 지역에서 "지역명 + 공항 픽업" 검색 시 엉뚱한 결과(페리 등)가 노출되는 오류를 방지하기 위해 검색어 주입 방식을 폐기.
  - 대신 사용자가 직접 공항/목적지를 입력할 수 있도록 Klook의 **"공항 교통편 전용 홈"** 및 **"렌터카 전용 홈"** 다이렉트 딥링크 버튼 2종으로 분리 교체.
  - 브랜드명 노출 대신 "공항 픽업 예약", "렌터카 검색" 등 직관적인 UI 텍스트로 변경.
- **기대 효과**:
  - 작동이 불완전한 위젯으로 인한 사용자 경험 저하를 방지하고 모바일 환경에서의 버튼 시인성과 터치 전환율 극대화.

### [Phase 8] 툴킷 필수 정보(도착 공항, 상세 여정, 공항 픽업) 강제 표시 최적화
- **작업 목적**:
  - AI 생성 내용 중 "공항 → 항구/목적지 이동" 등 특정 항목이 일부 여행지에서 누락되는 현상(할루시네이션) 방지.
  - 직항/경유 및 복잡도(is_complex)와 관계없이 여행의 큰 그림을 보여주는 '상세 여정 플래너'와 '공항 픽업'을 모든 지역에서 일관되게 제공하기 위함.
- **주요 구현 사항**:
  - `update-place-toolkit` 백엔드 프롬프트 개선: 
    - 기계적인 필수 작성 지시 대신, "도착 공항 명시: 복귀를 위한 최후의 보루", "상세 여정 플래너: 여행의 큰 그림 제공", "공항 픽업: 생판 모르는 곳에서의 첫 대면 대비"와 같이 사용자 관점의 당위성을 AI에게 설명하는 부드러운 형태로 프롬프트를 변경하여 안정적인 JSON 반환을 유도.
  - `src/components/PlaceCard/tabs/PlannerTab.jsx` UI 렌더링 분리: 
    - '상세 여정 플래너(`JourneyTimeline`)' 컴포넌트가 `is_complex`(복잡한 여행지) 조건문에 종속되어 보이지 않던 문제를 해결하기 위해, 복잡도 조건 외부로 컴포넌트를 분리하여 데이터만 존재하면 상시 렌더링되도록 조치.
  - **배포 이슈 대응**: 
    - Supabase Edge Function을 배포하는 과정에서 발생한 권한 에러(401 Unauthorized)를 방지하기 위해 `--no-verify-jwt` 옵션을 추가하여 배포 및 `.ai-context.md` 가이드라인 수정 완료.
- **기대 효과**:
  - 어떤 여행지를 검색하든 도착 공항부터 숙소 이동, 픽업 및 예상 경로 타임라인이 빠짐없이 노출되어, 플래너 탭이 진정한 스마트 트래블 가이드의 역할을 수행함.
