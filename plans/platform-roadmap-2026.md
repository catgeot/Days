# ✈️ 여행 플랫폼 중장기 로드맵 (웹 플래너 고도화 및 컴패니언 앱)

## 📌 비전 및 목표
- **웹 (계획/예약)**: 사용자가 탐색 페이지와 플래너 탭을 통해 필수 여행 정보와 교통/숙박 예약 링크를 원스톱으로 제공받는 환경 구축.
- **앱 (실행/가이드)**: 웹에서 수집된 여행 플래너 데이터를 바탕으로 사용자 타임라인에 맞춘 '맥락 인지형(Context-aware) 컴패니언 앱' 경험 제공.

---

## 🗺️ 전체 시스템 아키텍처 흐름

```mermaid
graph TD
    subgraph Web_Platform [웹: 여행 계획 및 예약 (In-app UX)]
        A[탐색 및 장소 카드 진입] --> B[플래너 탭 활성화]
        B -->|목표 여행지 데이터 전달| C[스마트 API 라우팅 엔진]
        C -->|동남아/아시아| D(Klook API)
        C -->|미주/글로벌| E(Mozio API)
        C -->|유럽/북미| F(Viator API)
        D & E & F --> G{결과 비교 및 일정 선택}
        G --> H[내 플래너에 담기]
        H --> I[최종 결제 시 파트너사 결제창 호출]
    end

    subgraph Database [통합 데이터베이스]
        H --> J[(사용자 여정 DB 구축)]
    end

    subgraph Mobile_App [모바일 앱: 타임라인 가이드]
        J --> K{백그라운드 스케줄러}
        K --> L[출발 전: 준비물, 면세점 알림]
        K --> M[출국/도착: 픽업 장소, 시간 안내]
        K --> N[체류 중: 맛집, 명소 큐레이션]
        K --> O[귀국: 픽업 안내, 귀국 팁]
    end
```

## 🗓️ 단계별 추진 계획 (Phase 1 ~ Phase 4)

### Phase 1: 플래너 내 필수 API 연동 기획 (웹 우선)
여행자가 가장 불편함을 느끼는 "현지 도착 직후의 이동" 문제를 최우선으로 해결합니다.

- **목표**: 2차 교통수단(픽업, 페리 등)의 예약 기능 구현
- **구현 대상**: 공항 ➔ 호텔 픽업, 공항 ➔ 항구 픽업, 페리 및 2차 교통수단
- **API 통합 수준 (핵심 UX)**:
  - 단순 아웃링크(다른 사이트로 튕겨나가는 방식)가 아닙니다.
  - 우리 플랫폼 플래너 탭 내부에서 검색, 차량 종류 확인, 옵션 선택까지 모두 자체 UI로 구현합니다.
  - **결제 분리**: 최종 "결제" 버튼 클릭 시에만 파트너사의 보안창을 호출합니다.

#### 🧠 지역 맞춤형 스마트 API 라우팅 (Smart API Routing & Zero-Search UX)
- **Zero-Search UX (입력 최소화)**: 사용자가 '장소 카드'에서 플래너를 열면, 목적지(공항 코드, 도시명)가 이미 확정되어 있습니다. 번거로운 타이핑 없이 날짜와 인원수 정도만 확인하면 곧바로 픽업 리스트를 볼 수 있습니다.
- **동적 API 라우팅**: 사용자의 목적지에 따라 가장 인벤토리가 많고 저렴한 API 파트너사를 선별 호출합니다. (예: 동남아시아 진입 시 Klook API 자동 호출)

#### 🖥️ 현재 플래너 UI 기반 인앱(In-app) 통합 방안
기존에 AI가 훌륭하게 생성하고 있는 플래너 화면에 인앱 결제/검색을 자연스럽게 녹여냅니다.
- **AI 꿀팁 텍스트 100% 보존**: "라부안 바조 공항(LBJ)에서 메인 항구까지 15분 소요"와 같이 기존 AI가 분석해주는 현지 정보와 꿀팁은 플랫폼의 핵심 가치이므로 현재처럼 텍스트로 상단에 유지합니다.
- **아웃링크의 점진적 위젯 전환 (Progressive Replacement)**:
  - 현재 카드 하단에 위치한 `[클룩 공항 픽업]`, `[12Go (아시아 교통)]` 같은 단순 외부 연결 버튼 영역을 **실시간 In-app 위젯**으로 교체합니다.
  - 사용자가 코모도 섬 장소 카드에 들어오면, 백엔드가 라부안 바조(LBJ) 코드를 Klook API에 전달하여 검색합니다.
  - 기존 버튼 자리에는 "🚗 스탠다드 세단 ₩15,000 [선택/담기]", "🚐 프라이빗 밴 ₩25,000 [선택/담기]" 형태의 실제 상품과 가격이 즉시 렌더링되어 이탈 없이 예약을 유도합니다.
- **단계적 개발**: Phase 1 대상인 '공항 픽업'과 '페리' 카드의 하단 영역만 먼저 위젯으로 교체해 성과를 측정한 뒤 항공, 숙박으로 점진적 확장을 진행합니다.

---

### Phase 2: 항공, 숙박 등 확장 API 제휴 기획 (웹 고도화)
- **목표**: 플래너 내에서 전체 여행 일정의 예약 및 탐색 통합.
- **API 통합 수준**: 자체 UI 내에서 검색 및 옵션 선택 완료 후 결제만 파트너사에 위임. 장소 카드 기반 목적지 자동 할당(Zero-Search UX) 로직 동일 적용.

---

### 💰 비즈니스 모델 및 API 비용 구조 (Business & Cost)
1. **API 호출 및 사용료는 '무료'입니다.**
2. **수익(Commission) 창출**: 사용자가 최종 결제를 완료하면 4% ~ 10%를 커미션으로 정산받습니다.
3. **인프라 비용 최적화**: API 호출 횟수 제한(Rate Limit) 방어를 위해 목적지에 맞는 API만 호출하는 스마트 라우팅과 자체 캐싱(Cache) 처리가 필수적입니다.

---

### Phase 3: 타임라인 기반 모바일 가이드 앱 상세 기획
웹에 축적된 '사용자 일정(Itinerary)' 데이터를 활용하는 스마트 컴패니언 앱.
- **핵심 정책**: 철저한 사용자 사전 동의 (위치, 푸시 알림, 타임존) 필수.
- **알림 기능**: 출발 전 준비물, 온라인 면세점, 출국 공항 정보, 현지 도착 픽업 알림, 체류 중 큐레이션, 귀국 안내 등.

### Phase 4: 모바일 앱 기술 스택 선정 및 개발
- **기술 스택**: `React Native` (프론트엔드), `FCM` 백그라운드 푸시 시스템.
- **주요 난제**: 도착지 타임존 자동 변환 로직, 비행기 탑승 시 오프라인 데이터 캐싱.

### Phase 5: 여행 정책 및 비자 자동화 모니터링 시스템 구축 (AI Travel Policy Monitor)
빠르게 변동하는 전 세계 입국 서류(e-Visa, ETA, ETIAS 등) 정책을 AI가 추적하여 사용자에게 가장 정확한 공식 링크와 정보를 제공하는 파이프라인.

- **1단계 (Data Extraction)**: 프론트엔드 하드코딩된 공식 비자/입국 서류 매핑 데이터를 통합 DB(Supabase)로 이관하여 즉각적 대응 환경 마련.
- **2단계 (AI Scheduled Crawler)**: 매주 주기적으로 Supabase Edge Function을 실행하여 외교부, IATA Timatic, 각국 이민국 공식 발표를 최신 검색 기반 AI(예: Perplexity API)로 모니터링.
- **3단계 (Alert & Auto-Update)**: 유럽 ETIAS 시행 등 정책 변경 감지 시 관리자(Slack)에 알림을 발송하고, 승인 시 DB 자동 업데이트 및 여행자 대상 뉴스레터 초안 생성.
