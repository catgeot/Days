# 스마트 트래블 툴킷 (Travel Toolkit) 최적화 계획 및 가이드라인

본 문서는 장소 카드의 '스마트 트래블 툴킷' 기능을 고도화하고 UI/UX 및 수익화(Affiliate) 링크 정확도를 개선하기 위한 다음 세션의 작업 명세서입니다.

---

## 1. 최적화 및 신규 개발 목표 (Next Session)

### 1-1. 헤더 레이아웃 (네비게이션 탭) 2열 구조 개편
- **이슈**: 모바일 및 데스크톱 환경에서 탭 개수(갤러리, 리뷰, 위키, 영상, 툴킷)가 증가함에 따라 1열 구조의 헤더(`PlaceChatPanel.jsx` 상단) 시인성이 저하됨.
- **해결 방안 (2열 헤더 구조 적용)**:
  - **1열 (상단)**: [뒤로가기] + [여행지 이름] + **[🎒 스마트 툴킷 (Killer Tab)]** + [북마크]
    - 서비스의 핵심(Killer) 기능인 툴킷 버튼을 여행지명과 함께 1열에 큼직하고 시인성 높게 단독 배치하여 클릭률을 극대화합니다.
  - **2열 (하단)**: [갤러리] | [리뷰] | [위키] | [영상]
    - 나머지 부가 기능 탭들을 2열에 넓게 분산 배치하여 모바일에서도 터치하기 편안한 UI/UX를 제공합니다.

### 1-2. 지역별 제휴 링크(Affiliate) 스마트 라우팅 및 세밀화
- **이슈**: 라로통가(Rarotonga) 등 소도시/오지의 경우 글로벌 범용 링크(예: Airalo 기본 검색)를 사용하면 잘못된 페이지나 검색 결과 없음으로 랜딩되는 문제 발생.
- **해결 방안**:
  - 대륙/국가/도시 데이터를 기반으로 **스마트 링크 제너레이터(Smart Link Generator)** 함수 구축 (`getLink` 함수 고도화).
  - 아시아(Klook 강세), 미주/유럽(GetYourGuide 강세) 등 대륙별 우선순위 파트너사 분기 처리.
  - 데이터에 정식 영문 국가명/도시명이 있을 경우, Airalo의 `airalo.com/cook-islands-esim` 등 국가별 직링크 규칙을 분석하여 하드코딩된 동적 파라미터 맵(Fallback Map)을 적용.

## 2. 사용자(운영자) 점검 및 작업 가이드라인 (스폰서 세팅)

툴킷의 제휴(Sponsored) 버튼에서 실제 수익을 창출하려면 운영자가 각 파트너사 프로그램에 가입하고 고유 ID를 발급받아야 합니다.

### [운영자 필수 작업 상세 가이드]
1. **각 파트너사 제휴(Affiliate) 프로그램 가입**:
   - 숙박: **Agoda Partners** 또는 **Booking.com Affiliate** 가입
   - 액티비티/교통: **Klook Affiliate** 및 **GetYourGuide 파트너** 가입
   - eSIM: **Airalo Partner / Impact** 플랫폼 가입
   - 항공권: **Skyscanner Partners** (또는 Travelpayouts 플랫폼 경유)
2. **제휴 고유 ID (Affiliate ID) 확보**:
   - 가입이 승인되면 `aid=123456`, `ref=gateokr` 등의 고유 파라미터를 발급받습니다.
3. **환경 변수(.env) 등록 준비**:
   - 코드에 ID를 직접 박아넣지 않도록, `.env` 파일에 `VITE_AGODA_AID`, `VITE_KLOOK_REF` 등의 변수를 추가하고 관리해야 합니다. 다음 세션에서 이 변수들을 연동할 예정이므로 발급된 키를 준비해 두세요.

### [서비스 점검 가이드라인]
1. **백엔드(Edge Function) 모니터링**:
   - 현재 `update-place-wiki`가 JSON을 응답하도록 변경되었습니다. Supabase Dashboard의 **Edge Functions 로그**를 확인하여 에러(파싱 에러, 타임아웃 등)가 없는지 점검하세요.
2. **모바일 사용성 (터치 영역)**:
   - 스마트폰에서 직접 접속하여 각 툴킷 카드의 버튼과 텍스트 크기가 읽기 편한지 점검합니다. 
3. **AI 데이터 환각(Hallucination) 점검**:
   - AI가 생성한 '지도 및 명소', '안전 정보' 등이 실제 현지 상황과 맞는지 여러 지역(대도시 1개, 오지 1개)을 비교 검토하세요.

## 3. 다음 세션 진행 순서 (계획)

- [ ] **Phase 1: 헤더 UI 리팩토링** (`PlaceChatPanel.jsx`) - 1열(장소+툴킷), 2열(나머지 탭) 구조로 레이아웃 재편.
- [ ] **Phase 2: 링크 제너레이터 고도화** (`ToolkitTab.jsx` 내부 로직 분리) - 장소 이름, 국가, 영문명을 기반으로 한 세밀한 URL 조합 유틸리티 생성.
- [ ] **Phase 3: 환경 변수 연동 기반 마련** - `import.meta.env`를 통해 제휴 ID를 안전하게 파라미터에 합성하는 로직 작성.
- [ ] **Phase 4: 예외 처리 (Fallback)** - 공식 사이트나 특정 파트너사 검색 결과가 없는 지역의 경우, 무작정 빈 검색 결과를 보여주기보다 구글 검색(`https://www.google.com/search?q=...`)으로 Fallback 하는 안전망 설계.
