# Phase 9-4: 탐색 모달(Search & Discovery) UX 최적화 분석 보고서

## 1. 교차 필터링(Cross-Filtering) 구조 분석

사용자의 제안인 **"테마 선택 시 대륙별 현황, 대륙 선택 시 테마별 현황 확인"**은 사용자가 탐색의 '축'을 직접 결정하게 하는 매우 훌륭한 접근 방식입니다. 이를 UI로 구현하기 위한 두 가지 방안을 분석했습니다.

### 방안 A: 기준 전환형 섹션 뷰 (추천)
가장 직관적으로 사용자의 요구사항을 충족하는 방식입니다.

*   **상단 토글:** `[ 대륙별로 보기 ]` | `[ 테마별로 보기 ]`
*   **시나리오 1 (테마별로 보기 선택):**
    *   **1차 탭:** 🏝️휴양 | 🌲자연 | 🏙️도심 | 🏛️문화 | 🧗모험
    *   '휴양' 탭 클릭 시 하단 콘텐츠:
        *   **아시아 (12개)** - [카드 리스트...]
        *   **유럽 (5개)** - [카드 리스트...]
        *   **오세아니아 (8개)** - [카드 리스트...]
*   **시나리오 2 (대륙별로 보기 선택 - 현재 기본값과 유사):**
    *   **1차 탭:** 아시아 | 유럽 | 북미 | 남미...
    *   '유럽' 탭 클릭 시 하단 콘텐츠:
        *   **도심 (15개)** - [카드 리스트...]
        *   **문화 (10개)** - [카드 리스트...]
        *   **자연 (4개)** - [카드 리스트...]

**분석:** 정보를 평면적인 리스트로 나열하지 않고, 2차 분류 기준으로 '섹션화(Grouping)'하여 보여주기 때문에 훨씬 입체적이고 정돈된 느낌을 줍니다. 모래 속 진주를 찾는 것이 훨씬 수월해집니다.

## 2. 초기 화면 큐레이션 결합 분석

큐레이션 기능과 필터링 기능을 한 화면에 자연스럽게 배치하기 위해서는 **'상태(State)'에 따른 UI 전환**이 필수적입니다.

### 2.1. 화면 상태 정의
1.  **초기 상태 (Home / Curation View):**
    *   검색어도 없고 필터도 선택되지 않은 모달을 처음 열었을 때의 화면입니다.
    *   **배치:**
        *   **히어로 배너 (Hero Section):** "이번 주말, 완벽한 휴양지 5선" (큰 이미지 기반 카드, 횡스크롤)
        *   **빠른 탐색 (Quick Links):** 동그란 아이콘 형태로 '테마별 탐색', '대륙별 탐색' 진입 버튼 제공.
        *   **인기 여행지:** 현재 인기순 Top 10을 간략히 표시.
2.  **탐색 상태 (Discovery / Filter View):**
    *   사용자가 '휴양' 테마나 '아시아' 대륙을 클릭한 상태입니다.
    *   초기 큐레이션 화면은 페이드아웃 되며, 위에서 분석한 **[방안 A: 기준 전환형 섹션 뷰]**로 화면이 전환됩니다.
3.  **검색 상태 (Search View):**
    *   상단 검색창에 텍스트를 입력하는 순간 작동하며, 모든 필터와 큐레이션을 덮고 정확한 텍스트 매칭 결과를 리스트 형태로 보여줍니다.

### 2.2. 장소 카드(Place Card) 디자인 개선
큐레이션과 섹션 뷰의 효율을 극대화하려면 카드의 디자인도 변경되어야 합니다.
*   **AS-IS:** 텍스트 위주 (이름, 영문명, 설명, 작은 태그)
*   **TO-BE 제안:**
    *   **이미지 도입:** 가능하면 대표 이미지(썸네일)를 카드 배경이나 좌측/상단에 배치하여 시각적 매력도 상승.
    *   **아이콘 배지:** 텍스트 태그 대신 직관적인 아이콘(🏝️, 🏙️)으로 공간 절약 및 인지성 향상.

## 3. 구현 단계 (Action Plan)

이러한 다각도 분석을 바탕으로, 코드를 수정할 때는 다음 순서로 접근하는 것이 안전하고 효과적입니다.

1.  **데이터 구조 정비:** `TRAVEL_SPOTS` 데이터가 대륙별/테마별로 쉽게 그룹화(Grouping)될 수 있도록 유틸리티 함수(useMemo 등) 작성.
2.  **UI 뼈대 수정:** SearchDiscoveryModal 내부에 3가지 상태(Curation, Filter, Search)를 렌더링하는 조건부 로직 구현.
3.  **필터링 컴포넌트(방안 A) 구현:** 대륙/테마 전환 토글 및 하위 섹션 렌더링.
4.  **큐레이션 컴포넌트 구현:** 더미 데이터 기반으로 매력적인 초기 화면 디자인.
5.  **카드 디자인 리팩토링:** 시각성이 강화된 새로운 카드 컴포넌트 적용.
