# Phase 8-6: 마이리얼트립(MRT) 파트너스 API 연동 및 링크 고도화 전략

## 1. 개요
기존 플래너 탭(스마트 트래블 툴킷)에 노출되던 단순 딥링크(에어비앤비, 기타 거절된 플랫폼)를 글로벌/국내 여행에 모두 강력한 인프라를 갖춘 **마이리얼트립(MyRealTrip)** 파트너스 생태계로 전면 교체합니다. 단순 버튼 교체를 넘어 지역별 타겟팅 전략이 반영된 **동적 상품 추천 위젯**을 동시 도입하여 전환율(CTR) 및 UX를 극대화합니다.

## 2. 지역별 맞춤 큐레이션 전략

| 지역 분류 | 주요 타겟 국가/도시 예시 | 핵심 전략 (추천 쿼리) | 위젯 노출 상품 유형 | 대체/추가 버튼 |
| :--- | :--- | :--- | :--- | :--- |
| **아시아/오세아니아** | 일본, 베트남, 태국, 대만, 호주 | 한국인 친화적 숙소 및 단독 투어 | `한인민박`, `현지 단독 투어` | 에어비앤비 → MRT 한인민박 검색 |
| **유럽/미주** | 프랑스, 영국, 이탈리아, 미국 | 복잡한 이동/입장 효율화 | `교통 패스`, `박물관/패스트트랙` | 교통/지도 섹션 내 MRT 패스 링크 |
| **특수 지역(중남미/아프리카)**| 코스타리카, 갈라파고스 등 | 인프라 부족 보완 (안전/이동) | `공항 픽업`, `자연 액티비티` | 픽업/투어 위주 MRT 검색 링크 |
| **국내** | 제주도, 부산, 강원도 | 렌터카 및 입장권 중심 | `렌터카`, `테마파크 입장권` | 렌터카/체험 중심 링크 |

## 3. 세부 구현 계획

### Step 1: 유틸리티 및 딥링크 구조 개선 (`src/utils/affiliate.js`)
- `getMultiLinks()` 함수 내의 하드코딩된 외부 링크(에어비앤비 등)를 정리.
- 마이리얼트립 전용 딥링크 생성 로직(검색 쿼리 기반) 구축.

### Step 2: 보안 프록시 API 생성 (`supabase/functions/fetch-mrt-products`)
- 브라우저 측 API 키 노출 방지를 위해 Supabase Edge Function 구축.
- 지역명과 카테고리를 Body로 받아 마이리얼트립 파트너스 API를 호출하고 결과를 JSON으로 프론트에 반환.

### Step 3: 커스텀 추천 위젯 컴포넌트 개발 (`MyRealTripWidget.jsx`)
- 대륙/지역 정보를 바탕으로 렌더링 시 최적의 카테고리 검색어를 자동 결정.
- Edge Function을 호출하여 상품 사진, 타이틀, 가격, 평점 등의 데이터를 가로 스크롤 캐러셀(가로 스와이프 UI) 형태로 렌더링.

### Step 4: `PlannerTab.jsx` 전면 통합
- 에어비앤비 및 유효하지 않은 버튼 제거.
- `JourneyTimeline`의 액션 버튼(공항 픽업, 숙소 검색, 페리 등)을 마이리얼트립 검색 결과로 우선 연결.
- 새로 만든 `MyRealTripWidget`을 UI 상의 적재적소(예: "🌴 현지 100% 즐기기" 섹션)에 삽입.

## 4. UI/UX 기대 효과
사용자는 플래너의 가이드를 읽는 동시에, 본인이 가는 지역의 특성에 가장 잘 맞는 **실제 상품 카드**를 시각적으로 확인하게 됩니다. 텍스트 링크보다 훨씬 높은 클릭 동기를 부여하며 여행 준비의 완결성을 높여줍니다.
