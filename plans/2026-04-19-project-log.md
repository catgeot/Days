# 2026-04-19 프로젝트 로그

## 작업 내역

### 1. 홈 지구본 FlyTo(플라이 투) UX 개선
* **문제점:** 기존 지구본 화면에서 줌인(고도 1.8 이하)된 상태일 때 마커 아이콘을 클릭해도 아무런 동작을 하지 않도록(FlyTo 방지) 하드코딩 되어 있어 일관된 UX를 제공하지 못함.
* **개선 사항:**
  * `src/pages/Home/components/HomeGlobe.jsx` 내 `flyToAndPin` 로직 수정.
  * 확대된 상태에서도 플라이 투 방지 로직을 제거하고, 클릭 시 항상 지정된 목표 고도로 비행하도록 일관성 확보.
  * `GLOBE_CAMERA_CONFIG`에 `FLY_TARGET_ALT` (2.1) 추가: 플라이 투 시 2.0 ~ 2.5 사이의 극적인 줌인 효과를 주도록 목표 고도를 2.1로 통일.
  * 자전 재개 고도(`AUTO_ROTATE_DISABLE_ALT`)를 기존 2.2에서 2.0으로 하향 조정하여, 2.1 고도에서도 플라이 투 완료 후 4초 대기 시 자연스럽게 자전이 재개되도록 설정.

### 2. 에디터스 픽(탐색창) 내 트립링크 동적 배너 연동 및 UI 최적화
* **문제점:** 기존 에디터스 픽(CurationSection)에 정적인 더미 카드(`PackageThumbnailCard`)만 배치되어 있었고, 트립링크의 동적 배너(iframe)를 연동할 때 728x90 형태의 가로형 배너가 세로형 카드 영역 내에서 심하게 깨지며 레이아웃 오류(오버플로우)를 발생시킴.
* **해결 및 개선 사항:**
  * **크기 및 규격 표준화:** 트립링크 관리자 페이지에서 300x250 배너를 생성하도록 유도하고, 코드(`tripLinkPackages.js`) 내에서도 해당 iframe 크기를 300x250으로 일치시킴.
  * **TripLinkIframeCard.jsx 신규 컴포넌트 개발:**
    * 기존 여행지 카드 디자인(모바일 240x320, PC 280x380)의 껍데기(둥근 테두리, 배경, hover 효과)를 그대로 유지함.
    * 카드 안쪽에서 300x250 크기의 iframe을 CSS `transform: scale()`을 이용해 카드 폭에 딱 맞게 축소(scale-0.75 ~ 0.9)하여 삽입. 트립링크 서버의 반응형 내부 스크립트를 깨뜨리지 않으면서도 전체 카드 디자인 컨셉과 이질감 없이 융화되도록 구현.
    * 'AD', '트립링크 특가' 등의 뱃지를 추가하여 플랫폼 관점에서의 표시광고법(공정위) 준수 조건 만족(플랫폼 UI 디자인의 퀄리티와 정체성을 유지).
  * **렌더링 분기 적용:** `CurationSection.jsx`에서 `promotedPackages` 데이터의 `type === 'iframe'` 여부에 따라 신규 동적 배너 카드가 2번째, 6번째 위치에 자연스럽게 렌더링되도록 수정.

### 3. 유럽 패키지 배너 추가 및 큐레이션 배치 최적화 (2026.04.19)
* **요구 사항:** 에디터스 픽에서 광고의 위치를 두번째, 여섯번째에서 연달아 노출되도록 변경하고, 장거리/유럽 카테고리에 적합한 패키지 상품을 추천하여 적용.
* **작업 내용:**
  * **배치 재조정 (`CurationSection.jsx`):** 기존 index 1, 5에 있던 배너 위치를 순수 여행지 카드 5장이 렌더링된 이후, 연속해서 6번째와 7번째 위치에 연달아 노출되도록 수정.
  * **장거리(유럽) 패키지 큐레이션 (`tripLinkPackages.js`):** '서유럽 핵심 일주(adKey: wx9egs)'와 '동유럽/발칸3국(adKey: 8zfodz)' 상품 배너를 `longhaul` 테마에 적용.
  * **휴양/올인클루시브 패키지 큐레이션 (`tripLinkPackages.js`):** '남태평양(adKey: 1c4mmw)'과 '코타키나발루/쿠알라룸푸르(adKey: nnpyr1)' 상품 배너를 `resort` 테마에 적용하고 큐레이션 타겟(`resortTargets`) 순위를 재정렬.
  * **크기 대응 (`TripLinkIframeCard.jsx`):** 트립링크에서 발급한 배너 규격이 336x280으로 확대됨에 따라, 모바일 기기(240px 카드)에서 잘림 현상이 없도록 scale 축소 비율을 기존 0.75에서 0.7로 하향 최적화.
  * **노출 순서 지능화 (`SearchDiscoveryModal.jsx`):** 기존에는 일일 무작위 셔플(`shuffled`) 배열의 순서가 그대로 유지되어 노출되었으나, 사용자의 검색 피로도를 낮추고 접근성을 극대화하기 위해 기획서(`targets`) 배열에 명시된 순서대로 추출 및 정렬되도록 추출 로직(`getSpotsByTargets`)을 개선. 이에 따라 파리, 로마, 오사카, 괌 등 인기도가 가장 높은 핵심 지역들이 항상 큐레이션 최앞단에 배치됨.
