# 모바일 UX 및 갤러리 로딩 전략 검토 보고서 (2026-04-04)

## 2. 모바일 UI/UX 추가 개선 사항

### 2.1 하이브리드 헤더 도입 (공간 효율화)
* **문제점**: 모바일 화면에서 [1열: 홈/검색바] + [2열: 대륙/테마 탭] + [3열: 세부 카테고리 탭]이 모두 상단에 고정(Sticky)될 경우, 실제 렌더링되는 본문(콘텐츠) 영역이 너무 좁아지는 현상 발생.
* **개선 방안 (스크롤 연동)**:
  * 모달 초기 진입 시: 기존처럼 전체 헤더 요소가 다 보임.
  * 스크롤 시(아래로): [1열: 홈/검색바]는 스크롤에 밀려 위로 사라짐.
  * 고정 영역(Sticky): [2열: 탭]과 [3열: 카테고리] 부분만 최상단에 고정되어 구조 파악을 돕고 넓은 본문 시야 확보.
  * 스크롤 시(위로): 다시 [1열] 영역이 나타나도록(헤더 숨김 토글) 구현 검토.

### 2.2 모바일 검색창 시인성 및 포커스 개선
* **문제점**: 다크모드 기반이라 검색바의 안내 텍스트(Placeholder)와 테두리 경계가 뚜렷하지 않음. 또한 모달이 열리자마자 자동으로 키패드가 올라와(focus) 화면을 가림.
* **개선 방안**:
  * 검색바 테두리 밝기 조정 및 배경색 명도 대비 강화.
  * `SearchDiscoveryModal.jsx` 진입 시 `inputRef.current.focus()`를 실행하던 초기 로직을 제거하여, 사용자가 **직접 검색창을 터치했을 때만** 키패드가 올라오도록 변경.

### 2.3 불필요한 컨트롤러 제거 및 아코디언 뷰 수정
* **좌우 화살표 제거**: 모바일 환경에서는 터치 스와이프가 직관적이므로, 큐레이션 섹션(`CurationSection.jsx`)에 노출되어 썸네일을 가리는 좌우 화살표(검은 바탕 버튼)를 모바일 뷰(`md:hidden`)에서 완전히 숨김 처리.
* **아코디언 토글 버그 수정**: 세부 카테고리 아코디언 메뉴의 닫기 기능이 작동하지 않는(열려만 있는) 문제 해결.
* **초기 상태 변경**: 특정 탭 진입 시 첫 번째 아코디언(예: 아시아)이 강제로 열려있는 로직을 폐기하고, 처음에는 전체 카테고리가 닫힌(접힌 리스트) 상태로 노출되도록 초기 상태(`selectedSubGroup = null`) 유지.

## 3. 이미지 로딩(Unsplash API) 최적화 검토

### 3.1 현재 상태 및 우려 사항
* **현재 방식 (Lazy Load)**: 세션 스토리지 폭발을 막기 위해 뷰포트에 카드가 들어올 때(`IntersectionObserver`)만 `usePlaceGallery` 훅이 트리거됨.
* **사용자의 우려**: 만약 한 명의 사용자가 스크롤을 계속 내려 200개의 여행지를 모두 훑는다면, 짧은 시간 안에 200건의 갤러리 검색 요청이 발생하여 Unsplash API의 시간당/일일 할당량(Quota)을 급격히 소진하게 되지 않는가?

### 3.2 최적화 및 방어 전략(Next Steps)
1. **백엔드 DB 캐싱(Supabase) 최우선 조회**:
   * 현재 `update-place-wiki` 엣지 함수나 `place_stats` 테이블에서 `gallery_data`를 저장하고 있다면, **클라이언트 측 Unsplash 직접 호출을 차단하고 100% DB만 바라보게 하는 방식(Read-Only)** 으로 전환 검토.
   * `SpotThumbnailCard` 같은 썸네일 컴포넌트에서는 고화질 여러 장이 아닌 대표 이미지(Index 0) URL 1개만 DB에서 바로 꺼내오는 전용 경량화 훅(`usePlaceThumbnail`)을 분리하여 생성.
2. **CDN / 썸네일 최적화**:
   * 원본 사이즈(regular) 대신 모바일 최적화 해상도(small) 이미지를 요청.
3. **요청 디바운스(Debounce) 및 로드 조건 강화**:
   * 단순히 뷰포트에 진입했다고 바로 API를 쏘는 것이 아니라, **화면에 머문 시간(예: 0.5초 이상)**을 측정하여 그냥 휙휙 지나가는 스크롤에는 API 요청을 보내지 않는(Cancel) 전략 추가.
