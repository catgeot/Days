# 2026-04-11 프로젝트 로그

## 🎯 오늘 진행한 주요 작업
1. **Travelpayouts 화이트라벨 연동 고도화**
   - 기존의 스크립트 기반 `WhiteLabelWidget`이 플래너 내부 스크롤과 충돌하는 UX 문제를 해결하기 위해 **모달(Modal) & iframe 연동 방식**으로 전면 개편.
   - 자체 서브도메인(`flights.gateo.kr`)을 활용한 CNAME 설정 방법을 안내하고 연동 완료.
   - 항공권 뿐만 아니라 **호텔(숙박)** 검색도 화이트라벨 모달을 통해 Travelpayouts 파트너 네트워크(Hotellook) 수익을 창출할 수 있도록 확장 구현 (`type` prop 활용).
   - **(추가)** `WhiteLabelWidget.jsx`의 과도한 가짜 검색 폼(Fake Form) UI 및 화려한 그라데이션 헤더 바를 제거하고, 플래너 탭의 다른 버튼들과 조화를 이루도록 깔끔한 파스텔 테마(항공: 하늘색, 숙박: 보라색)의 단일 버튼 디자인으로 개선.

2. **제휴 파트너 버튼 최적화**
   - 개별 제휴 승인이 반려된 아고다(Agoda), 부킹닷컴(Booking.com), 12Go 버튼 코드를 삭제 처리하여 사용자 혼란 방지. (숙박 검색은 Travelpayouts 화이트라벨 모달로 대체됨)
   - 클룩(Klook) 연동 시 방화벽(WAF) 차단 이슈를 확인하여, 검색어 동적 딥링크가 아닌 안전한 메인 홈 단축 링크(`tp.st`)로 롤백.
   - 글로벌 공항 픽업 커버리지 확대를 위해 **Intui.travel** 파트너사를 신규 연동 (사용자 마커 `712550`, 프로그램 ID `657`).
   - 플래너 탭 내에서 바로 픽업 출발지/도착지를 타이핑할 수 있도록 **Intui.travel 검색 폼 위젯(Search Widget)**을 컴포넌트화하여 탑재 완료.

3. **AI 할루시네이션 방지: 공식 비자/입국 서류 매핑 개선**

   - **(이슈 수정)** 필리핀 엘니도와 같은 여행지에서 AI 조언 텍스트(`data.advice`) 내의 "한국인 무비자" 문구 때문에 'K-ETA'가 오인 매칭되는 버그를 수정. 이제 AI 설명문구 제외하고 순수 **목적지 이름과 국가명만으로 엄격하게 공식 비자 사이트를 매칭**하도록 로직 개선.
