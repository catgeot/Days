# Mapbox 커스텀 기능 최대 활용 방안 및 구현 가이드

## 1. 개요
gateo.kr의 여행지 특성을 극대화하기 위해 Mapbox GL JS의 고급 기능을 활용하는 방안입니다. 단순한 지도 표시를 넘어, 사용자에게 **시각적인 탐험 경험(Cinematic Experience)**과 **실용적인 정보 탐색 기능**을 제공하는 것을 목표로 합니다.

## 2. 맵박스 기능 최대 활용 가이드라인

### A. 시네마틱 'Fly To' 경험 (우주에서 목적지까지)
*   **개념**: 위키 탭 지도 초기 진입 시 바로 목적지를 보여주지 않고, 전체 지구본 뷰(`projection: 'globe'`)를 먼저 렌더링. 중앙에 '여행 출발(Play)' 버튼을 제공하여, 클릭 시 지구본에서 목적지까지 날아가는 극적인 `flyTo` 애니메이션 연출.
*   **작동 방식**:
    *   **초기 상태**: Zoom 레벨 1~2, 지구본 형태 렌더링. 안개(Fog) 효과 적용으로 우주 공간 느낌 부여.
    *   **재생 버튼**: 지도 한가운데 Play 버튼 오버레이. 클릭 시 애니메이션 시작.
    *   **Fly To 애니메이션**: `map.flyTo({ center: [lng, lat], zoom: 12, pitch: 60, bearing: -20, duration: 8000 })` 적용. 천천히 대륙을 지나 목적지 지형이 3D로 솟아오르는 연출.
    *   **중간 정지(Skip/Stop)**: 애니메이션 도중 화면을 터치/드래그하거나 우측 하단 'Skip' 버튼을 누르면 즉시 애니메이션을 종료하고 목적지로 `jumpTo` 또는 해당 위치에서 사용자가 제어권을 획득(`map.stop()`).

### B. 목적지 특성별 지도 스타일 다변화 (Style Switcher)
여행지의 성격(대자연 vs 대도시)에 따라 최적의 지도 스타일을 선택할 수 있는 버튼 컨트롤 제공.
*   **Outdoors (현재 사용중)**: 등고선, 산책로, 지형이 잘 보여 **자연경관(스위스, 국립공원 등)**에 적합.
*   **Satellite-Streets (위성 + 도로)**: 실제 위성 사진 위에 도로망이 겹쳐 보여 휴양지나 독특한 지형을 리얼하게 볼 때 적합.
*   **Streets / Navigation**: **대도시(뉴욕, 런던 등)**에서 골목길, 상호명, 대중교통 라인을 명확히 볼 때 적합.
*   **3D Buildings (도시 특화)**: 대도시 줌 인 시 건물이 3D 입체로 솟아오르도록 `fill-extrusion` 레이어 추가. (현재 3D Terrain과 조합하여 도시의 스카이라인 감상 가능).

### C. 실용적 상호작용 (Interactive Layers)
*   **POI (관심 지점) 토글**: 여행객에게 필수적인 명소, 식당, 카페, 대중교통 역 등의 아이콘 레이어를 버튼으로 껐다 켤 수 있는 기능.
*   **낮/밤 (Light/Dark) 토글 또는 일조량 시뮬레이션**: Mapbox는 태양의 위치를 시뮬레이션할 수 있습니다. 시간에 따라 그림자가 변하는 효과를 주거나, UI 테마에 맞게 다크 모드 지도로 전환.

## 3. 구현 설계 (PlaceMiniMap.jsx 변경 방안)

### 단계 1: 지구본 프로젝션 및 안개(Fog) 설정
*   `Map` 컴포넌트 속성에 `projection="globe"` 추가.
*   `fog={{ range: [0.8, 8], color: '#242b4b', 'horizon-blend': 0.5 }}` 속성 추가로 지구본 외곽의 우주/대기권 효과 연출.

### 단계 2: 상태 관리 및 오버레이 UI 구축
*   **상태 변수 추가**:
    *   `mapState`: 'idle'(초기 대기), 'playing'(비행 중), 'ready'(도착 후 자유 탐색)
    *   초기 뷰 상태(Initial View State)를 `[lng, lat]`가 아닌 `zoom: 1.5` 중심 좌표로 설정.
*   **오버레이 UI**: `mapState === 'idle'` 일 때 지도를 덮는 반투명 오버레이와 Play(▶) 버튼.
*   **Skip 버튼**: `mapState === 'playing'` 일 때 우측 하단에 'Skip' 버튼 렌더링.

### 단계 3: 애니메이션 제어 로직
*   `playFlyTo` 함수: `mapRef.current.flyTo({...})` 호출.
*   이벤트 리스너: `onMoveEnd` 이벤트를 감지하여 애니메이션 완료 시 `mapState`를 'ready'로 변경.
*   중단 로직: 지도를 클릭/드래그하거나 Skip 버튼 클릭 시 `mapRef.current.stop()` 호출 후 `mapState` 갱신.

### 단계 4: 부가 컨트롤 추가
*   좌측 상단 2D/3D 버튼 외에 **[위성 지도 / 지형 지도]** 토글 버튼 추가.
*   (옵션) 3D 빌딩 레이어 활성화 함수 작성.

## 4. 사이트 특성(gateo.kr)에 맞춘 추천 기능
1. **스토리텔링형 도입부**: 매거진 형식의 위키 탭에 가장 잘 어울립니다. 사용자가 글을 읽기 전 지도를 통해 '여행지로 떠나는 느낌'을 시각적으로 먼저 체감하게 합니다.
2. **도시 vs 자연 스마트 스타일**: 데이터(categories, denseRegion 등)를 활용해 대자연(산맥)인 경우 Terrain 중심으로, 도심인 경우 3D Building과 Street 중심으로 초기 렌더링 스타일을 분기하는 것도 훌륭한 고도화 방안입니다.
