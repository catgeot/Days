# 트립링크 누락 목적지 추가 및 핫딜 배너 팝업 기획

## 1. 트립링크 링크 구조 파악 및 활용 방안

트립링크에서 제공하는 3가지 방식에 대한 서비스 적용 전략은 다음과 같습니다.

*   **단축 URL (`https://info.triplink.kr/a/...`)**
    *   **설명**: 사용자를 목적지로 바로 이동시키는 순수 링크입니다.
    *   **우리 서비스 적용 방식 (추천 1순위)**: 기존 서비스의 UI/UX 일관성을 유지하기 위해 **가장 권장하는 방식**입니다. 홈 화면 인피드 큐레이션 카드(`PackageThumbnailCard`)나 버튼에 이벤트로 연동하여, 배너 이미지나 카드 디자인은 gateo.kr 스타일에 맞게 직접 구성하고 클릭 시 해당 단축 URL로 새 창 띄우기를 합니다.
*   **iframe (`<iframe src='...' />`)**
    *   **설명**: 트립링크에서 만들어둔 디자인 배너를 그대로 웹 페이지 내에 삽입하는 방식입니다.
    *   **우리 서비스 적용 방식**: 특정 영역(예: 웹뷰, 사이드바 빈 공간)에 광고를 그대로 노출할 때 유용합니다. 하지만 우리 서비스의 반응형 UI 및 모달 스크롤과 충돌할 여지가 있어, 메인 큐레이션보다는 독립적인 공간(예: 홈 우측 상단 팝업)에 제한적으로 사용하기 적합합니다.
*   **이미지 링크 (`<a href='...'><img src='...' /></a>`)**
    *   **설명**: iframe을 사용할 수 없는 환경을 위해, 트립링크 서버에 있는 이미지를 불러오고 클릭 시 단축 URL로 이동하게 하는 방식입니다.
    *   **우리 서비스 적용 방식**: iframe보다 레이아웃을 제어하기 수월하므로, 트립링크의 배너 디자인을 그대로 쓰고자 할 때는 iframe 대신 이미지 링크 방식을 사용하는 것이 React 환경에서 더 깔끔합니다.

**✅ 결론 및 진행 방식**: 
1.  **홈 탐색창 인피드 큐레이션**: 우리 디자인 컴포넌트(`PackageThumbnailCard`) + **단축 URL**
2.  **홈 우측 상단 핫딜 배너 팝업**: 팝업 틀은 우리가 제작하고 내부 컨텐츠는 **단축 URL + 제공 이미지(이미지 링크 방식)** 또는 커스텀 이미지로 개발.

## 2. 누락된 타겟 목적지 추가 계획

홈 화면 테마별 큐레이션 배열(`familyTargets`, `longhaulTargets`, `resortTargets`)에 선언된 목적지 중 `travelSpots.js` DB에 없는 항목 10개를 신규 추가합니다.

*   **가족/효도 누락**: 다낭, 삿포로, 오사카, 후쿠오카, 칭다오(청도 포함)
*   **장거리/유럽 누락**: 두브로브니크
*   **휴양/올인클루시브 누락**: 사이판, 푸꾸옥, 코타키나발루, 호놀룰루(하와이는 이미 있으나 별도 주도로 추가 고려)

**작업 내용**:
- `travelSpots.js`에 위 10개 장소에 대한 객체(좌표, 설명, 카테고리 등) 신규 작성.

## 3. 홈 우측 상단 핫딜 배너 팝업 개발 계획

트립링크의 긴급 모객 특가, 골프 패키지 등을 홍보하기 위한 플로팅 팝업을 구현합니다.

*   **컴포넌트**: `src/pages/Home/components/HotDealBanner.jsx` 신규 생성.
*   **노출 위치**: PC 우측 상단 (z-index 최상위), 모바일에서는 하단 탭바 위 또는 화면 중앙 팝업 형태 고려.
*   **기능 요건**:
    *   트립링크 단축 URL 클릭 이벤트 연동.
    *   우측 상단 `X` 버튼 제공.
    *   **"오늘 하루 보지 않기"** 기능 (로컬 스토리지 `hide_hotdeal_banner` 키 활용, 자정 기준 초기화).
*   **연동 로직**: `HomeUI.jsx` (또는 메인 화면 적절한 계층) 최상단에 렌더링되도록 삽입.
