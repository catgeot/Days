# 트립링크 패키지 데이터 파이프라인 및 관리 기획

패키지 상품은 단일 어트랙션(예: 디즈니랜드 입장권)과 달리 **"출발일", "모객 수", "수명주기(판매 기간)"**가 존재하므로, 단순 하드코딩으로는 관리가 불가능합니다. 따라서 제휴 링크 데이터를 어떻게 수집하고 노출할지에 대한 명확한 전략이 필요합니다.

## 1. 링크 노출 방식의 3가지 옵션

### 옵션 A. 정적 카테고리 딥링크 (수명주기 무한) - Phase 1 추천
특정 "상품"이 아닌, 트립링크의 **"지역별 패키지 메인 화면"**이나 **"상시 기획전"**으로 연결하는 링크입니다.
- **장점**: 수명주기가 없으므로 한 번 연동하면 유지보수(링크 갱신)가 필요 없습니다.
- **단점**: 고객에게 뾰족한 단일 상품 가격이나 매력을 바로 보여주지 못해 클릭률이 다소 떨어질 수 있습니다.
- **데이터 구조**: 프론트엔드(`constants.js`) 또는 DB에 지역 키워드 맵핑 테이블만 유지.

### 옵션 B. API / 상품 피드(XML/CSV) 연동 (수명주기 자동화) - Phase 2 필수
트립링크 측에서 제공하는 제휴 API나 상품 피드 데이터를 스케줄러(Supabase Edge Function 또는 Cron Job)를 통해 매일 주기적으로 긁어와(Sync) 우리 DB에 저장하는 방식입니다.
- **장점**: 판매가 종료되었거나 출발일이 지난 상품을 자동으로 내리고, 최신 가격과 썸네일을 보여줄 수 있습니다.
- **단점**: 트립링크 측에서 API나 피드를 제공해야 하며, 개발 공수(데이터 파싱 로직, 스케줄러)가 발생합니다.

### 옵션 C. 어드민(CMS) 기반 수동 관리 (수명주기 수동 제어)
"이달의 특가"처럼 소수의 강력한 패키지 상품만 관리자가 DB에 직접 등록하고, 노출 시작일과 종료일(수명주기)을 설정하는 방식입니다.
- **장점**: 마케팅적으로 가장 밀고 싶은(수익률이 좋은) 상품을 전략적으로 노출할 수 있습니다.
- **단점**: 관리자가 주기적으로 만료된 링크를 내리고 새 링크를 찾아 등록해야 하는 휴먼 리소스가 발생합니다.

---

## 2. 데이터 구조 설계 (Supabase DB 기반)

동적 상품(옵션 B 또는 C)을 노출하기 위해 Supabase에 아래와 같은 테이블을 신설하는 것을 제안합니다.

### 테이블명: `affiliate_packages`
| 컬럼명 | 타입 | 설명 |
|---|---|---|
| `id` | uuid | PK |
| `region_keyword` | text[] | 타겟 지역 키워드 (예: `['유럽', '파리', '로마']`) |
| `title` | text | 노출할 패키지 배너/상품명 |
| `image_url` | text | 상품 썸네일 이미지 URL |
| `affiliate_url` | text | 트립링크 제휴 트래킹 딥링크 |
| `price` | int | 패키지 시작가 (옵션) |
| `start_date` | timestamp | 배너 노출 시작일 (수명주기 시작) |
| `end_date` | timestamp | 배너 노출 종료일 (수명주기 만료) |
| `is_active` | boolean | 강제 비활성화 여부 |

## 3. 추천하는 진행 플랜 (데이터 관점)

1. **Step 1 (현재 상태 파악)**: 트립링크 제휴 센터(어드민)에 접속하여 **"API를 제공하는지"**, 아니면 **"카테고리 딥링크만 발급 가능한지"** 확인해야 합니다.
2. **Step 2 (Phase 1 적용)**: 만약 개별 상품 API 연동이 복잡하다면, 우선 **옵션 A(지역별 카테고리 기획전 링크)**로 플래너 탭을 덮어서 작동 로직을 검증합니다.
3. **Step 3 (고도화)**: 트립링크에서 핫한 개별 상품 링크를 주기적으로 제공해준다면, `affiliate_packages` DB를 구축하여 **수명주기(end_date)가 지나면 프론트엔드에서 자동으로 노출되지 않도록 처리**합니다.

지금 당장 가장 필요한 것은 **트립링크 제휴 대시보드에서 어떤 형태의 데이터/링크를 추출할 수 있는지 확인**하는 것입니다. 상품 피드 API가 있나요? 아니면 수동으로 링크를 복사해야 하는 구조인가요?
