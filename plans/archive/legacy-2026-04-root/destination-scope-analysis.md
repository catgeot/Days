# 여행지 수량 스코프 분석

## 🎯 핵심 질문

**"세계인이 여행하는 일반 수준의 여행지는 몇 개나 될까?"**

## 📊 주요 여행 플랫폼 벤치마크

### 1. 글로벌 여행 가이드
```
Lonely Planet (론리플래닛)
├─ Top 500 Places: 전 세계 주요 여행지
├─ Best in Travel: 매년 50개 추천
└─ 실제 가이드북 커버리지: 약 200-250개 주요 목적지

National Geographic Traveler
├─ Best Trips: 약 200개 추천 여행지
├─ 대륙별 분류
└─ 자연/문화/도시 균형 배치

Rick Steves (유럽 전문)
├─ 유럽 주요 도시: 약 100개
├─ 소도시 포함: 약 200개
└─ 패키지 투어 제공 도시: 50개
```

### 2. 여행사 패키지 범위

#### 대형 온라인 여행사 (OTA)
```
Expedia / Booking.com
├─ 주요 패키지 도시: 500-1,000개
├─ 실제 인기 목적지: 약 200-300개
└─ 상위 20%가 전체 예약의 80% 차지

Klook / KKday (아시아 중심)
├─ 커버리지: 약 400개 도시
├─ 실제 인기 도시: 약 100-150개
└─ 액티비티 제공 도시: 200-250개
```

#### 국내 여행사
```
하나투어 / 모두투어
├─ 패키지 여행 제공: 약 100-150개 도시
├─ 항공권 + 호텔 자유여행: 약 200-300개 도시
└─ 실제 판매 상위 도시: 50-80개

여행박사 / 인터파크 투어
├─ 전체 상품 도시: 150-200개
└─ 매출 상위 도시: 30-50개
```

### 3. 항공사 취항지
```
대한항공 / 아시아나
├─ 직항 도시: 약 60-80개
├─ 코드쉐어 포함: 약 200-300개
└─ 실제 주요 노선: 30-50개

저비용 항공사 (LCC)
├─ 동남아 중심: 30-50개
├─ 일본/중국: 20-30개
└─ 총 커버리지: 60-80개
```

## 💡 실무적 고려사항

### 1. 누락되면 안 되는 여행지 (Red Flag)
```
✅ 필수 포함 (Tier 1 - 50개)
- 파리, 런던, 로마, 뉴욕, 도쿄, 방콕...
- 유네스코 세계유산 주요 지점
- 7대 불가사의 관련 도시
- 국가별 수도 및 주요 관광 도시

❌ 초기 누락 가능 (Phase 2 추가)
- 소도시 (예: 프랑스의 아비뇽, 콜마르)
- 신흥 여행지 (예: 조지아, 알바니아)
- 극소수 오지 (예: 소코트라, 케르겔렌)
```

### 2. 지구본 vs 카테고리 트리 분리 전략

#### 지구본 표시 기준 (80-100개)
```javascript
const shouldShowOnGlobe = (destination) => {
  return (
    destination.tier === 1 || // 항상 표시
    (destination.tier === 2 && !isInDenseRegion(destination)) || // 밀집 지역 아니면 표시
    (destination.tier === 3 && isVisuallyInteresting(destination)) // 시각적 흥미
  );
};

// 예시
파리: showOnGlobe = true (Tier 1)
암스테르담: showOnGlobe = false (파리와 가까워 밀집)
남극: showOnGlobe = true (Tier 3이지만 시각적 흥미)
```

#### 카테고리 트리 전체 표시 (200-300개)
```
모든 여행지가 트리에는 포함
→ 검색 가능
→ 대륙/국가별 분류
→ 북마크 가능
→ 상세 정보 제공

지구본에 없어도 문제없음
→ "암스테르담"을 검색하면 트리에서 찾을 수 있음
→ 클릭 시 지도에서 해당 위치로 Fly To (마커는 안 보여도)
```

### 3. 데이터 업데이트 주기
```
Tier 1 (50개): 분기별 검토 (3개월)
├─ 주요 정보 변경 모니터링
└─ AI 툴킷 재생성

Tier 2 (120-150개): 반기별 검토 (6개월)
├─ 인기도 변화 추적
└─ 신규 여행지 추가/교체

Tier 3 (30-100개): 연간 검토 (12개월)
├─ 트렌드 반영
└─ 커뮤니티 피드백 반영
```
