## 작업 내역

### 1. 홈 지구본 FlyTo(플라이 투) UX 개선
* **문제점:** 기존 지구본 화면에서 줌인(고도 1.8 이하)된 상태일 때 마커 아이콘을 클릭해도 아무런 동작을 하지 않도록(FlyTo 방지) 하드코딩 되어 있어 일관된 UX를 제공하지 못함.
* **개선 사항:**
  * `src/pages/Home/components/HomeGlobe.jsx` 내 `flyToAndPin` 로직 수정.
  * 확대된 상태에서도 플라이 투 방지 로직을 제거하고, 클릭 시 항상 지정된 목표 고도로 비행하도록 일관성 확보.
  * `GLOBE_CAMERA_CONFIG`에 `FLY_TARGET_ALT` (2.1) 추가: 플라이 투 시 2.0 ~ 2.5 사이의 극적인 줌인 효과를 주도록 목표 고도를 2.1로 통일.
  * 자전 재개 고도(`AUTO_ROTATE_DISABLE_ALT`)를 기존 2.2에서 2.0으로 하향 조정하여, 2.1 고도에서도 플라이 투 완료 후 4초 대기 시 자연스럽게 자전이 재개되도록 설정.

### 2. 에디터스 픽(탐색창) 내 트립링크 동적 배너 연동 및 UI 최적화
* **문제점:** 기존 에디터스 픽(CurationSection)에 정적인 더미 카드(`PackageThumbnailCard`)만 배치되어 있었고, 트립링크의 동적 배너(iframe)를 연동할 때 728x90 형태의 가로형 배너가 세로형 카드 영역 내에서 심하게 깨지며 레이아웃 오류(오버플로우)를 발생시킴.
* **해결 및 개선 사항:**
  * **크기 및 규격 표준화:** 트립링크 관리자 페이지에서 300x250 배너를 생성하도록 유도하고, 코드(`tripLinkPackages.js`) 내에서도 해당 iframe 크기를 300x250으로 일치시킴.
  * **TripLinkIframeCard.jsx 신규 컴포넌트 개발:**
    * 기존 여행지 카드 디자인(모바일 240x320, PC 280x380)의 껍데기(둥근 테두리, 배경, hover 효과)를 그대로 유지함.
    * 카드 안쪽에서 300x250 크기의 iframe을 CSS `transform: scale()`을 이용해 카드 폭에 딱 맞게 축소(scale-0.75 ~ 0.9)하여 삽입. 트립링크 서버의 반응형 내부 스크립트를 깨뜨리지 않으면서도 전체 카드 디자인 컨셉과 이질감 없이 융화되도록 구현.
    * 'AD', '트립링크 특가' 등의 뱃지를 추가하여 플랫폼 관점에서의 표시광고법(공정위) 준수 조건 만족(플랫폼 UI 디자인의 퀄리티와 정체성을 유지).
  * **렌더링 분기 적용:** `CurationSection.jsx`에서 `promotedPackages` 데이터의 `type === 'iframe'` 여부에 따라 신규 동적 배너 카드가 2번째, 6번째 위치에 자연스럽게 렌더링되도록 수정.

### 3. 유럽 패키지 배너 추가 및 큐레이션 배치 최적화 (2026.04.19)
* **요구 사항:** 에디터스 픽에서 광고의 위치를 두번째, 여섯번째에서 연달아 노출되도록 변경하고, 장거리/유럽 카테고리에 적합한 패키지 상품을 추천하여 적용.
* **작업 내용:**
  * **배치 재조정 (`CurationSection.jsx`):** 기존 index 1, 5에 있던 배너 위치를 순수 여행지 카드 5장이 렌더링된 이후, 연속해서 6번째와 7번째 위치에 연달아 노출되도록 수정.
  * **장거리(유럽) 패키지 큐레이션 (`tripLinkPackages.js`):** '서유럽 핵심 일주(adKey: wx9egs)'와 '동유럽/발칸3국(adKey: 8zfodz)' 상품 배너를 `longhaul` 테마에 적용.
  * **휴양/올인클루시브 패키지 큐레이션 (`tripLinkPackages.js`):** '남태평양(adKey: 1c4mmw)'과 '코타키나발루/쿠알라룸푸르(adKey: nnpyr1)' 상품 배너를 `resort` 테마에 적용하고 큐레이션 타겟(`resortTargets`) 순위를 재정렬.
  * **크기 대응 (`TripLinkIframeCard.jsx`):** 트립링크에서 발급한 배너 규격이 336x280으로 확대됨에 따라, 모바일 기기(240px 카드)에서 잘림 현상이 없도록 scale 축소 비율을 기존 0.75에서 0.7로 하향 최적화.
  * **노출 순서 지능화 (`SearchDiscoveryModal.jsx`):** 기존에는 일일 무작위 셔플(`shuffled`) 배열의 순서가 그대로 유지되어 노출되었으나, 사용자의 검색 피로도를 낮추고 접근성을 극대화하기 위해 기획서(`targets`) 배열에 명시된 순서대로 추출 및 정렬되도록 추출 로직(`getSpotsByTargets`)을 개선. 이에 따라 파리, 로마, 오사카, 괌 등 인기도가 가장 높은 핵심 지역들이 항상 큐레이션 최앞단에 배치됨.
  * **렌더링 최적화 (`TripLinkIframeCard.jsx`):** 사용자가 횡스크롤로 접근할 때 배너가 뒤늦게 로딩되며 스피너가 보이거나 빈 화면이 노출되는 UX 저하를 방지하기 위해, 기존의 `IntersectionObserver` 지연 로딩 로직을 전면 제거. 모달 진입 시 iframe이 브라우저 네이티브 지연 로딩(`loading="lazy"`)과 함께 즉시 렌더링되도록 수정하여 체감 로딩 속도를 비약적으로 단축.

### 4. 홈 탐색 모달(`SearchDiscoveryModal`) UI 시인성 개선 및 배너 조화 (2026.04.19 - Session 2)
* **문제점 및 요구사항:** 
  * 기존 `TripLinkIframeCard`의 남색/검정 배경이 주변의 밝은 해변/하늘 사진을 담은 여행지 카드들과 이질감이 있음.
  * 패키지 상품을 가리키는 `CurationSection` 상단 안내 버튼이 우측 끝에 있어 찾기 어려움.
  * 가로 스크롤 영역에서 좌우 네비게이션 버튼과 스크롤바가 다크 모드에 묻혀 시인성이 떨어지고 현재 위치 파악이 어려움.
* **해결 및 개선 사항:**
  * **배경색 조화 최적화:** `TripLinkIframeCard.jsx`의 배경을 청명한 하늘색에서 시작하여 하단의 모래/흙 톤으로 떨어지는 그라데이션(`from-sky-400 via-sky-200/80 to-[#d4a373]`)으로 변경. 실제 해변/휴양지 사진 카드와 유사한 느낌을 주어 리스트 전체의 조화로움을 대폭 향상.
  * **맥락형 버튼 파싱:** `CurationSection.jsx`의 `subtitle` 내부에 포함된 핵심 문구("휴양 패키지 추천" 등)를 파싱하여, 텍스트 중간에 인라인 칩 버튼 형태로 삽입. 글을 읽는 맥락 안에서 자연스럽게 클릭을 유도하고, 누를 시 배너 위치로 횡스크롤(`scrollToAd`) 되도록 UX 개선.
  * **스크롤 시인성 강화:** `SearchDiscoveryModal.jsx`의 글로벌 `.custom-scrollbar` 두께를 16px로 키우고 밝기를 0.6~0.9로 대폭 상향하여 시인성 확보. `CurationSection.jsx`의 좌우 화살표 버튼을 눈에 띄는 화이트/글래스톤(`bg-white/90`)으로 변경하고, 크기 확대 및 위치 조정을 통해 배경 카드와 혼동되지 않도록 개선.
  * *(보류/제거)*: 카드 사이에 추가하려던 `PackageGuideCard`는 디자인 재검토를 위해 렌더링을 보류하고 광고 배너 위치를 기존(index 5)으로 환원.

### 5. 트립링크 네이티브 썸네일 카드 및 대화면 모달 팝업 구현
* **요구 사항:** 기존 iframe을 작은 카드에 넣는 방식 대신, 여행지 카드와 동일한 디자인의 썸네일을 만들고 클릭 시 대화면 모달을 띄우는 방식으로 전환.
* **작업 내용:**
  * **TripLinkSectionCard.jsx 신규 개발:**
    * 패키지의 대표 키워드(예: Da Nang, Paris)를 이용해 `usePlaceGallery` 훅으로 Unsplash 사진을 배경으로 불러옴.
    * `SpotThumbnailCard`와 동일한 디자인 구조(둥근 테두리, 그라데이션 오버레이, 하단 텍스트 영역) 적용.
    * 상단에 "특가 패키지" 뱃지, 우측 상단에 "AD" 표시로 광고임을 명확히 표시.
    * `IntersectionObserver`를 이용한 지연 로딩 적용으로 성능 최적화.
  * **TripLinkModal.jsx 신규 개발:**
    * 썸네일 카드 클릭 시 화면 중앙에 띄워지는 대화면 모달 (최대 90vh, max-width 1024px).
    * 상단 헤더에 "gateo x 트립링크 특가 패키지" 제목 및 "새 창으로 보기" 버튼 배치.
    * iframe 영역에 충분한 여유 공간(p-6~p-16) 확보하여 답답함 해소.
    * 배경 클릭 또는 X 버튼으로 모달 닫기 기능 구현.
    * `document.body.style.overflow` 제어로 모달 열릴 때 배경 스크롤 방지.
  * **tripLinkPackages.js 데이터 구조 확장:**
    * 각 패키지에 `targetKeyword` (Unsplash 검색용), `title` (카드 제목), `description` (카드 설명) 속성 추가.
    * 배너 크기를 모두 1024x768로 통일하여 모달에서 최적 렌더링.
  * **CurationSection.jsx 수정:**
    * `TripLinkIframeCard` import를 `TripLinkSectionCard`로 교체.
    * `onSelectPackage` prop 추가 및 카드 클릭 시 콜백 연결.
  * **SearchDiscoveryModal.jsx 수정:**
    * `TripLinkModal` import 추가.
    * `selectedPackage` state 추가 및 `handlePackageSelect` 핸들러 구현.
    * 3개 큐레이션 섹션(`family`, `longhaul`, `resort`)에 모두 `onSelectPackage` prop 전달.
    * 모달 오픈 시 `selectedPackage` 초기화 로직 추가.
    * 하단에 `{selectedPackage && <TripLinkModal ... />}` 조건부 렌더링 추가.
  * **제휴 주체 수정:**
    * 기존 "gateo x 노랑풍선"을 "gateo x 트립링크"로 수정하여 정확한 어필리에이트 파트너 명시.
