# Gemini Text 모델을 활용한 사이트 내 콘텐츠 보강 및 지도/장소 연동 계획 (Action Plan)

## 1. 개요
현재 임베딩 모델을 도입하는 대신, 서비스가 이미 사용 중인 **Gemini 2.5 Flash / Pro 계열(텍스트 생성/Function Calling)**과 프론트엔드 라우팅 기법을 활용하여 사이트 내 부족한 "실제 지도 진입점", "주변 맛집/명소" 콘텐츠를 신속하게 채워 넣는 방안입니다. 

---

## 2. 단계별 구현 계획

### Step 1: 기본 지도 연동 버튼 신설 (No AI, 즉시 적용 가능)
가장 빠르고 확실한 방법으로, 기존 장소 데이터를 활용해 구글 지도 및 구글 여행 등 외부 서비스로 곧바로 이어지는 진입점을 추가합니다.

- **위치**: `PlaceCard` 컴포넌트 내부(예: 기본 요약 뷰 또는 장소 정보 패널 하단)
- **방식**: 
  - 현재 로드된 장소의 좌표(`lat`, `lng`)나 장소명(`name`)을 활용하여 동적으로 URL 링크(a 태그) 생성.
  - 구글 맵 진입: `https://www.google.com/maps/search/?api=1&query={lat},{lng}` 또는 `https://www.google.com/maps/search/?api=1&query={name}`
  - **UI/UX 추가**: "[📍 구글 지도에서 보기]" 같은 명시적인 버튼을 두어 사용자가 실제 맵으로 편하게 이동할 수 있게 합니다.

### Step 2: "로컬 왓슨" 정보 고도화 및 주변 맛집/명소 큐레이션 (Gemini API 프롬프트 수정)
현재 `update-place-wiki` 백엔드 함수가 호출하는 프롬프트(AI가 장소 백과 정보를 가져오는 부분)를 보강하여, 실제 주변 맛집과 명소에 대한 실용적인 정보를 **JSON 형태**로 더 많이 받아오도록 수정합니다.

- **프롬프트 변경 사항**:
  - 기존: 장소의 기본 정보, 팁, 가는 방법 등을 생성.
  - **추가**: "이 장소 근처의 현지인 추천 맛집 2곳과 함께 가볼 만한 주변 명소 2곳의 이름과 한 줄 설명, 그리고 구글 지도에서 검색할 수 있는 검색 키워드(현지어/영어)를 JSON에 함께 담아줘."
- **DB 업데이트 (`place_wiki` 테이블)**: 
  - `ai_curation` 또는 `nearby_spots`와 같은 JSONB 컬럼을 활용하거나 `info` JSON 내부에 `nearby_spots` 객체를 포함시킵니다.
- **UI 렌더링 (PlaceWikiDetailsView.jsx 등)**: 
  - AI 백과사전 탭 내부에 **[🍽️ 주변 로컬 맛집]** 및 **[📸 함께 방문하기 좋은 곳]** 영역을 신설.
  - 각 항목 클릭 시 **Step 1**과 연계하여 해당 장소의 구글 맵스 검색결과 창을 새 탭으로 엽니다.

### Step 3: 장소 백과 내 "연관 검색 키워드(Hashtag)" 동적 라우팅
장소 백과에서 제미나이가 알려준 팁이나 특징(예: `#에펠탑야경`, `#도쿄라멘`)을 클릭하면, 사이트 내부 기능과 연동시킵니다.

- **방식**:
  - 제미나이에게 장소 요약과 함께 3~4개의 테마 해시태그를 생성하도록 요청.
  - 사용자가 해시태그를 클릭하면 내부 `DailyReport` (로그북) 전체 검색(`?q=도쿄라멘`)으로 넘어가서, 다른 유저들의 해당 주제 관련 여행기를 찾아볼 수 있게 라우팅.

### Step 4: 챗봇(PlaceChat)을 활용한 동적 가이드 제공
장소 챗봇(`PlaceChatPanel`) 내에서 사용자가 "여기 근처 맛집 알려줘"라고 물어봤을 때의 대응 능력을 향상시킵니다.

- **방식**:
  - `usePlaceChat`의 프롬프트를 강화: "사용자가 맛집이나 주변 장소를 물어보면, 항상 실제 가게 이름과 함께 구글 맵 검색을 제안해라."
  - 챗봇 응답이 장소명을 담고 있다면, 정규식이나 마크다운 파싱을 통해 자동으로 해당 단어를 외부 검색 맵으로 이어지는 `<a target="_blank">` 링크 컴포넌트로 변환.

## 3. 요약 및 우선순위 (마일스톤)

| 단계 | 작업 내용 | 난이도 | 영향도 |
| --- | --- | --- | --- |
| **Phase 1 (즉시)** | 장소 카드 내 [📍 구글 지도에서 보기] URL 동적 버튼 추가 | 낮음 | 매우 높음 (사용성 즉각 개선) |
| **Phase 2 (단기)** | Edge Function 프롬프트 수정하여 "주변 맛집/명소" 데이터 JSON 추가 수집 및 렌더링 | 중간 | 높음 (AI 정보의 실용성 증가) |
| **Phase 3 (중기)** | AI 백과 해시태그와 로그북 내부 리뷰 검색 연동 & 챗봇 응답 내 지도 링크 파싱 | 중간 | 중간 (사이트 체류 시간 및 탐색 연계 증가) |

이러한 계획은 임베딩이나 별도의 복잡한 파이프라인(Vector DB 구축) 없이 현재의 Supabase + Gemini Text 모델 구조에서 바로 프론트엔드 및 Edge Function 수정만으로 달성할 수 있습니다. 승인해 주시면 [Phase 1]부터 바로 코드를 반영하겠습니다.
