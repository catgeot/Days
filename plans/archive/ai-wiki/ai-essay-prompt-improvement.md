# AI 감성 에세이 작성 고도화 기획안

## 1. 현재 문제점 분석

사용자가 제공한 "보라카이 처남 식구들 여행" 결과물을 보면 다음과 같은 문제점이 명확합니다:

1. **환각 및 입력 데이터 무시 (Hallucination)**
   - 입력값: "처남 식구들과 함께"
   - 출력값: "오롯이 저 자신에게 집중", "혼자만의 시간을 보내며"
   - 원인: 프롬프트에 `원본 메모의 사실관계와 감정선은 절대 훼손하지 말고`라는 지시가 있지만, "감성적인 에세이"라는 페르소나와 결합되면서 AI가 상투적인 혼자만의 고독한 여행 템플릿을 무의식적으로 적용해버렸습니다.

2. **클리셰 남발 (Stiff & Artificial Tone)**
   - "시간이 멈춘 듯한 그곳에서", "모든 것이 완벽한 조화를 이루고 있었습니다", "그 자체로 한 폭의 그림 같았습니다" 등 너무 전형적이고 기계적인 AI식 미사여구가 가득합니다. 사람이 실제로 쓰는 담백하거나 재치 있는 표현이 결여되어 있습니다.

3. **시각 자료(사진)와 텍스트의 부자연스러운 연결**
   - 현재 프롬프트는 첨부된 사진의 개수만 넘길 뿐, 사진의 실제 분위기나 내용을 에세이 흐름에 유기적으로 녹여내지 못하고 단절된 문단을 생성하고 있습니다.

## 2. AI 모델 검토 및 추천

현재 에세이 생성에 사용 중인 모델은 `src/pages/DailyReport/hooks/useLogbookAI.js`를 확인한 결과 `gemini-2.5-pro`입니다. 

*   **현재 상태**: `gemini-2.5-pro`는 문장력 자체는 훌륭하지만, 지시어가 너무 추상적이면("서정적이고 감동적인") 방대한 학습 데이터 중 가장 흔한 패턴(진부한 에세이 클리셰)을 꺼내오는 경향이 있습니다.
*   **권장 사항 (모델 변경)**: 글쓰기, 특히 창의적이고 사람 같은 톤앤매너(Human-like Tone)와 뉘앙스 처리에는 **`gemini-3.1-pro`** (또는 최신 `gemini-3.1-pro-preview`)를 적극 권장합니다. 
    *   3.1 버전은 인간의 감정선이나 미묘한 디테일을 살리는 데 훨씬 탁월하며, 진부한 표현(AI 냄새)을 덜 씁니다.
    *   또한, 제공된 메모의 키워드(예: 처남 식구들)를 누락하거나 왜곡하지 않고 충실하게 반영하는 인스트럭션 추종 능력이 개선되었습니다.
*   `useLogbookAI.js` 내의 모델 호출을 `gemini-3.1-pro` (또는 `.ai-context.md`에 명시된 사용 가능 모델)로 교체하는 것을 제안합니다.

## 3. 프롬프트 개선안 (`src/pages/Home/lib/prompts.js`)

기존 프롬프트를 전면 개편하여, "사람 냄새가 나는(Human-written)" 글이 되도록 구체적인 룰을 추가합니다.

**[기존 프롬프트]**
> 당신은 사람들의 마음을 울리는 감성적인 여행 에세이 작가입니다. ... 한 편의 서정적이고 감동적인 여행 수필을 작성해주세요. 문체는 정중하고 차분한 독백체(~했습니다, ~였습니다, ~더군요)를 사용하세요. 메모와 사진에 담긴 장소의 분위기를 풍부한 시각적 묘사로 살려주세요. ...

**[개선 프롬프트 적용 방향]**
1.  **AI 클리셰 금지어 설정**: "시간이 멈춘 듯한", "한 폭의 그림", "완벽한 조화" 등의 뻔한 표현을 명시적으로 금지합니다.
2.  **데이터 무결성(Fact-Check) 최우선**: 사용자가 입력한 동행인, 날씨, 감정 등의 '팩트'를 절대 임의로 바꾸지 않도록 강력히 지시합니다. (예: 동행인이 있는데 혼자 갔다고 쓰면 안 됨).
3.  **톤앤매너 재정의**: '서정적이고 감동적인'이라는 추상적 지시 대신, **"브런치(Brunch) 작가나 유명 에세이스트처럼 담백하면서도 관찰력이 돋보이는"** 문체로 지시하여 세련도를 높입니다.
4.  **시각적 묘사 구체화**: 오감(시각, 청각, 후각 등)을 활용한 디테일한 묘사를 요구합니다.

이러한 개선안을 바탕으로 Code 모드에서 실제 프롬프트 코드와 모델 버전을 업데이트하겠습니다.
