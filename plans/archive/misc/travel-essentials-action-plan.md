# Gateo.kr 여행자 필수 정보 큐레이션 및 하이브리드 수익화(Monetization) 실행 계획

## 1. 배경 및 비전
기존 검색 포털이나 여행사 사이트의 "과도한 광고성 노이즈"에 지친 여행자들을 위해 **특정 지역/여행지별 필수 정보(지도, 비자, 필수 앱, 항공/숙박 등)만 선별 제공**한다는 초기 비전을 유지합니다.
그러나 지속 가능한 서비스 운영 및 AI API 비용, 고품질 서버 유지를 위해 **사용자 경험을 해치지 않는(Non-intrusive) '신뢰 기반의 하이브리드 수익화 전략'**을 결합하여, 궁극적으로 서비스 질을 향상시키는 선순환 구조를 구축합니다.

---

## 2. 필수 정보(Travel Essentials) 8대 카테고리 정의 및 수익화 모델(Affiliate) 결합 방안

AI(Gemini)를 활용하여 각 장소(국가/도시 단위)별 핵심 데이터를 구조화(JSON)하여 수집하되, 제휴 마케팅(Affiliate) 링크를 사용자가 불쾌감을 느끼지 않는 선에서 스마트하게 배치합니다.

1. **지도/명소 (Map & POI)**: 광고 없는 순수 평점 기반 로컬 맛집 및 필수 명소 (구글 맵 검색 연동) - *비수익(신뢰도 구축용)*
2. **사전 비자 (Visa/K-ETA/ESTA)**: 무비자 여부, 대행사 배제 및 **공식 관공서 홈페이지 링크 제공** - *비수익(신뢰도 구축용)*
3. **교통/렌터카 (Transport/Rental)**: Klook, GetYourGuide 등 **공식 제휴 API/링크**를 통한 교통 패스(JR패스, 유레일 등) 안내 (가격 투명성 보장) - *수익 가능(제휴 링크)*
4. **필수 앱 (Must-have Apps)**: 국가별 특화 앱 가이드 - *비수익*
5. **공항 픽업 / 유심 / eSIM**: 클룩(Klook), 마이리얼트립, 에어알로(Airalo) 등 여행자 필수 구매 항목을 **제휴 링크(Affiliate)**로 삽입. "가장 빠르고 저렴한 수단"이라는 큐레이션 가치 제공 - *수익 모델 핵심*
6. **항공권 (Flight)**: 스카이스캐너 제휴 파트너십 API를 통한 장소 기반 직행 검색 링크 - *수익 가능(제휴 링크)*
7. **숙박 (Accommodation)**: 부킹닷컴, 아고다 Affiliate 링크 연동. 단순 배너가 아닌, **AI가 추천하는 "해당 도시의 최적 숙박 위치(예: 오사카는 우메다 근처)" 팁**과 함께 검색 버튼 제공 - *수익 모델 핵심*
8. **안전/비상 (Safety)**: 영사콜센터, 현지 긴급 번호 등 - *비수익*

> **핵심 전략 (The "Trust & Convert" Strategy)**
> 수익화(항공/숙박/eSIM) 기능과 비수익 공익 정보(비자 공식사이트/안전/현지 지도)를 5:5 비율로 혼합(Hybrid)하여 배치. 사용자는 "비자 대행사 사기 조심해라, 여기 공식 홈페이지로 가라"는 공익적 조언을 보고 플랫폼을 깊이 신뢰하게 되며, 이어진 호텔/유심 링크에서도 거부감 없이 자발적으로 구매로 이어지게 됩니다.

## 3. 구체적인 실행 계획 (UI/UX & 백엔드 설계)

### Phase 1: AI 프롬프트 재설계 및 DB 스키마 확장 (백엔드)
`update-place-wiki` 엣지 펑션 내 프롬프트를 고도화하여 정보 분류와 제휴 코드 삽입 기반을 마련합니다.

1. **Supabase DB 테이블 확장 (`place_wiki`)**
   - `essential_guide` JSONB 컬럼 구조화.
2. **Edge Function 프롬프트 설계**
   - AI에게 "여행자 생존 키트" 페르소나를 부여하되, **수익화가 불가능한 항목(비자, 일반 앱, 지도)은 공식 정보만을**, **수익화가 가능한 항목(숙박 위치, 유심, 교통 패스)은 실질적 조언(Advice)만을 생성**하도록 분리합니다.
   - 예: AI가 "도쿄 숙박은 신주쿠나 시부야가 교통이 편리함" 이라는 텍스트를 JSON으로 주면, 프론트엔드가 자체적으로 보유한 아고다 제휴 파라미터를 붙인 링크 버튼을 동적으로 생성하여 결합.

### Phase 2: "스마트 트래블 툴킷 (Travel Toolkit)" UI 구현 (프론트엔드)
노이즈(배너 광고, 깜빡이는 팝업)를 완전히 배제하고, 애플(Apple) 스타일의 깔끔한 카드 그리드/대시보드 UI를 적용합니다.

1. **PlaceCard 탭 신설 (`[ 🧳 여행 툴킷 ]`)**
   - **"클린 UI" 규칙 적용**: 구글 애드센스 같은 디스플레이 광고(배너) 삽입 절대 금지. 정보 카드 자체가 버튼 역할을 하도록 디자인.
2. **Affiliate 버튼의 시각적 투명성**
   - 제휴 링크가 포함된 카드(숙박, eSIM, 교통 패스 등)는 하단에 작게 `Sponsored` 또는 `파트너사 제휴 링크로, 사이트 운영에 큰 도움이 됩니다`라는 문구를 투명하게 명시하여 오히려 팁을 얻어가는 유저의 호의를 자극(팁 문화).
   - 비자/관공서 카드는 `Official` 또는 `공식 기관` 마크를 달아 신뢰성 극대화.

### Phase 3: 파트너십/제휴(Affiliate) 프로그램 순차적 도입
사이트 트래픽과 운영 로드에 맞춰 단계적으로 파트너십을 연결합니다.

1. **단기**: Agoda / Booking.com (숙박), Klook (교통/패스), Airalo (eSIM) 파트너 프로그램 가입 및 범용 URL 파라미터(Affiliate ID) 삽입 방식 적용.
2. **중기**: 스카이스캐너 파트너 API 연동하여 장소 카드 내에 '현재 항공권 대략적 최저가'를 텍스트로 보여주고 클릭 유도.
3. **장기**: 클릭률/전환율(CTR/CVR) 데이터를 Supabase에 로깅하여, 어떤 도시에서 어떤 종류의 툴킷(교통 vs 숙박 vs 유심)이 많이 클릭되는지 분석하고 AI 프롬프트를 더욱 최적화.

## 4. 진행 순서 및 결론

1. **[기획]** 수익/비수익 항목 분리 설계 완료 (본 문서)
2. **[디자인]** `PlaceCard` 내 툴킷 탭 컴포넌트 개발 (광고 배너 느낌을 없앤 대시보드 형태의 클린 UI)
3. **[백엔드]** `update-place-wiki` 엣지 펑션 프롬프트 수정하여 8종 필수 정보 JSON 추출 로직 반영
4. **[인프라]** 각종 여행 파트너(Affiliate) 프로그램 가입 후 발급받은 ID를 환경변수(`.env`) 처리하여 URL 동적 조합 로직 구현.

이러한 **하이브리드 전략**은 사용자에겐 "상업적 노이즈 없는 진정한 비서"로 느껴지게 하면서도, 실제로는 가장 높은 전환율(호텔, 유심 등 필수재)을 발생시켜 향후 AI 모델 운영 및 서비스 고도화 비용을 충당할 수 있는 가장 현실적이고 강력한 돌파구가 될 것입니다.
