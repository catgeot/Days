# 지구본 렌더링 성능 최적화 및 데이터 병합 계획서

## 1. 목표
- `travelSpots.js`의 마커 개수를 30개에서 60개로 확장하여, 5개 섹터별로 전 세계 6대륙에 균형 있게 배치.
- 텍스트 라벨과 겹치는 마커 데이터 중복 제거.
- `HomeGlobe.jsx` 내 줌인 시 발생하는 DOM 프레임 드랍 문제 원천 차단 (진정한 LOD 적용).

## 2. 작업 상세 내용

### 2.1. 데이터 통합 및 승격 (`travelSpots.js` & `citiesData.js`)
- **현재 상황**: 5개의 카테고리(휴양, 자연, 도시, 문화, 모험)에 대륙당 1개씩, 총 30개의 마커가 존재합니다.
- **수정 목표**:
  - `citiesData.js` 내에 있는 각 대륙별 주요 명소를 1개씩 추가로 발탁하여 각 카테고리당 12개, **총 60개의 마커로 확장**합니다.
  - 마커로 승격된 **총 60개의 장소는 `citiesData.js` 파일에서 완전히 삭제**합니다.
  - 이를 통해 줌인 시 텍스트 라벨과 아이콘 마커가 시각적으로 겹치거나, 불필요한 메모리 중복이 발생하는 현상을 완벽히 차단합니다.

### 2.2. DOM 부하 원인 해결 및 LOD 적용 (`HomeGlobe.jsx`)
- **현재 상황**: 줌인 시 텍스트만 보이도록 하기 위해 `lodLevel === 1`일 때 CSS(`opacity: 0 !important; pointer-events: none;`)로만 HTML 마커를 감추고 있습니다. 이 방식은 브라우저 DOM 트리에서 노드가 계속 계산되어 줌을 조작할 때 심각한 프레임 드랍을 유발합니다.
- **수정 목표 (True LOD)**:
  - `htmlElementsData` 속성(prop) 값 자체에 LOD 조건을 겁니다.
  - `lodLevel > 0`일 때 React 상태에서 빈 배열(`[]`)을 전달하도록 코드를 변경합니다.
  - **수정 예시**: `htmlElementsData={lodLevel > 0 ? [] : (isZenMode ? [] : allMarkers)}`
  - DOM 노드 자체가 언마운트되므로 줌인 상태에서의 부하가 극적으로 감소합니다.
