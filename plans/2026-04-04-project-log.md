### 2.5. 모바일 탐색 모달 네이티브 스크롤 기반 하이브리드 헤더 적용 및 갤러리 로딩 방어 로직 (추가)
*   **문제**: 모달에서 아래로 스크롤 시 검색 헤더가 공간을 너무 많이 차지하고 스크롤 튕김 현상이 발생했으며, 빠른 스크롤 시 썸네일 카드 갤러리 API 호출이 낭비되는 이슈 파악.
*   **해결책**:
    *   **iOS 주소창 버그 방어**: `SearchDiscoveryModal` 최상단에 `fixed inset-0` 대신 `fixed top-0 left-0 w-full h-[100dvh]`를 적용하여 모바일 브라우저 주소창 토글에 따른 뷰포트 밀림 현상 완벽 해결.
    *   **하이브리드 헤더 도입**: JS 기반의 인위적 스크롤 감지 이벤트(`handleScroll` 내부의 상태 제어)를 완전히 제거하고 CSS 기반 네이티브 스크롤 레이아웃으로 전환. 1열(홈/검색창)은 스크롤 컨테이너에 포함시켜 스크롤을 내리면 자연스럽게 화면 밖으로 밀려 올라가도록 구현하고, 2열과 3열은 `sticky top-0`으로 그룹화하여 안정적으로 상단 고정. 튕김(반동) 현상 원천 차단.
    *   **모바일 3열 탭 구조 통합**: 모바일에서만 노출되던 아코디언 메뉴(`AccordionGroup.jsx`)를 완전히 폐기하고, 사용자의 요청에 따라 3열을 가로 스크롤 가능한 탭 UI로 통합. 직관성 극대화.
    *   **갤러리 API 디바운싱**: `SpotThumbnailCard`의 IntersectionObserver에 0.5초 대기열(`setTimeout`)을 추가하여 스크롤 패스(빠르게 지나가는 썸네일)에는 API를 호출하지 않도록 캐싱 방어 로직 강화.
    *   **오버레이 최적화 및 텍스트 효과**: 사진 전체를 덮던 어두운 오버레이를 하단 50% 영역에만 강하게 적용(`bg-gradient-to-t from-black/90 via-black/40 to-transparent`)하여 원본 사진의 밝은 명암을 최대한 보존하면서도 텍스트(Drop-shadow) 가독성 향상.
*   **관련 커밋**: `163daf6` (feat(ux): 모바일 탐색 모달 네이티브 스크롤 기반 하이브리드 헤더 적용 및 갤러리 로딩 방어 로직 구현)

### 2.6. PC 플로팅 아일랜드 UI 도입 및 모바일 탭 시인성 강화 (사용자 피드백 반영)

*   **해결책**:
    *   **vh 리사이즈 방어**: 모달 최상단에 `h-[100dvh]` 대신 `fixed inset-0`을 적용하여 모바일 브라우저 주소창 토글 시 화면 튕김(커짐/작아짐) 현상 완벽 해결.
    *   **PC 갤러리 룩 (Fluid UI)**: 왼쪽 사이드바를 화면에 꽉 차게 붙이는 대신, 여백과 둥근 모서리(`rounded-3xl`), 유리 질감(`backdrop-blur-2xl`) 섀도우를 가진 **플로팅 아일랜드(Floating Island)** 형태로 분리. 우측 장소 카드들의 여백(Gap)을 시원하게 늘리고, Hover 시 스케일 업 및 3D 틸트(`duration-1000`) 효과를 부여해 역동성 확보.

### 2.7. 검색 모달 AI 오타 보정 및 지오코딩 기능 복구 및 UX 개선
*   **문제**: 검색 모달(`SearchDiscoveryModal`)이 분리되면서, 입력창에 데이터에 없는 여행지나 오타를 입력하고 엔터를 쳤을 때 기존에 작동하던 AI 오타 보정 프록시(`fetchProxyGemini`) 및 Mapbox 지오코딩 로직이 호출되지 않는 문제(연결 유실) 발생. 또한, 모바일 환경에서 검색어 입력 시 좌측에 나타나는 연관 검색어(Related Tags)가 3D 지구본을 가리는 UX 이슈.
*   **해결책**:
    *   **onSearch Prop 연결**: `HomeUI.jsx`에서 모달 컴포넌트로 `onSearch` prop을 내려주고, 모달 내부의 `<input>` 태그에서 `onKeyDown` (Enter 키) 발생 시 `handleSmartSearch`가 정상 호출되도록 복구 완료.
    *   **AI 탐색 유도 뷰 (UX 개선)**: 실시간 필터링 결과가 없을 때 나타나던 '검색 결과가 없습니다' 텍스트를 제거하고, "엔터(Enter) 키를 누르시면 AI가 전 세계 지도를 검색하여 즉시 해당 위치로 비행합니다!"라는 명확한 콜투액션(Call To Action)을 포함한 매력적인 애니메이션 UI로 대체.
    *   **검색어 찌꺼기 완벽 제거**: 모달이 열릴 때마다 무조건 검색어를 빈칸으로 초기화(`setQuery('')`)하고, 부모 컴포넌트(`HomeUI`)의 더미 검색바의 `inputValue` 찌꺼기도 제거하여 검색어 겹침 현상 원천 차단.
    *   **모바일 연관 검색어 숨김**: 모바일 환경에서는 화면이 좁으므로 좌측 툴팁 형태의 연관 검색어를 숨겨(`hidden md:flex`) 지구본 뷰포트를 온전히 보존.
*   **관련 커밋**: `b20df6a` (feat(search): 검색 모달 AI 오타 보정 지오코딩 복구 및 UX 개선) 외 1건

*   **해결책**:

    *   **모바일 줌 제한 해제**: 기존 모바일에서 줌을 멀리서 막아두던 `controls.minDistance = isMobile ? R * (1 + 1.5) : R * 1.01` 로직을 PC와 동일하게 `R * 1.01`로 통일해 자유로운 탐색 지원.

### 2.9. 딥링크 방식 변경에 따른 대규모 SEO 최적화 및 동적 사이트맵 적용
*   **문제**: 사이트의 주소 체계가 파라미터 방식에서 딥링크(예: `/place/seoul`) 방식으로 크게 변경됨에 따라, 구글 검색 엔진이 기존 색인을 잃고 404 에러를 발생시킬 위험이 높았음. 또한 탭 전환 시(`?tab=gallery`) 파라미터가 변경되며 중복 콘텐츠로 인식될 우려가 존재.
*   **해결책**:
    *   **동적 sitemap.xml 자동 생성**: `vite.config.js`를 수정하여, 빌드 시 `travelSpots.js` 내 약 200여 개의 기본 여행지 데이터를 불러와 `/place/{slug}` 형태의 동적 라우트들을 사이트맵에 굽도록 `vite-plugin-sitemap` 연동.
    *   **Canonical 태그 및 동적 메타태그 적용**: `react-helmet-async` 라이브러리를 도입하고 공통 `<SEO />` 컴포넌트를 개발. `App.jsx`를 Provider로 감싼 뒤 `PlaceCard`가 마운트될 때마다 주소의 `tab` 파라미터를 무시하고 순수한 딥링크(`/place/{slug}`)를 `rel="canonical"`로 설정하여 SEO 파워를 한 곳으로 응집.
    *   **크롤링 로봇 제어 (robots.txt)**: `public/robots.txt`를 생성하여 구글 봇에게 `sitemap.xml`의 위치를 알려주고, `/*?tab=*` 경로의 무분별한 크롤링을 차단(Disallow)하여 서버 부하 및 중복 색인 문제 해결.
*   **관련 커밋**: `feat(seo): 적용 딥링크 최적화, 동적 사이트맵 및 캐노니컬 메타태그 구현`
