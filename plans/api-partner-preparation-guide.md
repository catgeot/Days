# 🚀 여행 플랫폼 API 연동을 위한 사전 준비 가이드

Phase 1(교통/픽업) 개발을 본격적으로 시작하기 위해, 플랫폼 운영자(대표님)께서 직접 준비하고 승인받아야 할 파트너사 가입 및 API 발급 절차를 안내해 드립니다.

---

## 1. 🔑 필수 API 파트너사 가입 및 키 발급 신청

단순한 '제휴 링크(Affiliate)'가 아닌, 우리 사이트 내에서 검색을 띄우기 위한 **'Open API' 권한**을 얻는 것이 핵심입니다.

### ① Klook (클룩) Partner / Affiliate Network (추천 1순위)
아시아권 픽업 및 페리 인벤토리 확보를 위해 가장 먼저 가입해야 합니다.
- **가입 링크**: [Klook Affiliate Program](https://affiliate.klook.com/)
- **신청 절차**: 
  1. 위 링크에서 파트너 계정 가입
  2. 트래픽 출처(현재 우리 플랫폼 도메인) 등록
  3. **핵심**: 가입 승인 후, 담당자(Account Manager)에게 문의하거나 개발자 센터를 통해 **"Open API (Server-to-Server) 연동 권한"**을 요청해야 합니다. 단순 위젯/배너가 아닌 API 연동임을 명시하세요.
- **요청 사항**: 테스트용(Sandbox) API Key, 프로덕션 API Key, API 명세서(Documentation).

### ② Mozio (모지오) B2B API
전 세계 지상 교통수단(공항 픽업, 셔틀) 전용으로 구조가 매우 깔끔합니다.
- **가입 링크**: [Mozio For Partners](https://www.mozio.com/en-us/partners/)
- **신청 절차**:
  1. 위 사이트에서 "Partner with us" 또는 "API Access" 신청 폼 작성.
  2. 플랫폼의 성격(여행 플래너)과 월 예상 트래픽(또는 향후 계획) 기재.
  3. 승인 후 Sandbox(테스트) API Key 즉시 발급 가능.

### ③ 기타 대안 파트너사
- **Viator (Tripadvisor 계열)**: [Viator Partner Program](https://www.viator.com/partner-resources/api) - 구미주권 타겟 시 추가 가입.
- **MyRealTrip (마이리얼트립)**: 마이리얼트립 파트너스는 B2B API를 선별적으로 제공하므로, 공식 제휴 이메일을 통한 직접 컨택이 필요합니다.

## 2. 📄 재무 및 행정 준비 서류 (개인 자격 가입 가능)

커미션을 정산받고 위젯을 발급받기 위해 필요한 행정 절차입니다. 결론부터 말씀드리면 **초기 런칭 시에는 사업자가 없어도 무방합니다.**

- **개인(Personal) 자격 가입 (초기 런칭 시 권장)**
  - Klook, Agoda 등의 어필리에이트 프로그램은 여행 블로거나 유튜버, 개인 개발자도 가입할 수 있도록 열려 있습니다.
  - 사업자 등록증 없이 **개인 신분증(여권 사본)**과 **개인 명의 외화 통장(달러 수취 가능 계좌)**만 있으면 가입 및 화이트 라벨 위젯 발급, 수익금 정산이 모두 가능합니다.
- **기업(Business) 자격 가입 (향후 고도화 시)**
  - 추후 플랫폼 매출이 커져 세금 처리가 필요해지거나, 단순 위젯을 넘어 복잡한 자체 API 연동(B2B Server-to-Server) 심사를 받아야 할 때 영문 사업자 등록증(Business License)과 기업용 외화 계좌로 업그레이드하시면 됩니다.

## 3. 💡 대안: 화이트 라벨(White-Label) 및 Iframe 위젯 도입
API 연동의 개발 공수를 최소화하고 즉시 서비스 런칭을 원하신다면 **'화이트 라벨'** 방식이 가장 현실적이고 훌륭한 대안입니다.

- **방식**: Klook이나 Mozio, Agoda 등은 파트너를 위해 완성된 검색/예약창을 HTML Iframe 형태(화이트 라벨 위젯)로 제공합니다.
- **장점**: 
  - 개발 공수가 거의 제로(Zero)에 가깝습니다. UI를 직접 그릴 필요 없이 파트너사가 만들어둔 예약 엔진이 우리 플랫폼 안에 깔끔하게 심어집니다.
  - 심사 절차가 까다로운 Open API 발급을 기다릴 필요 없이, 제휴 가입 즉시 코드를 발급받아 붙여넣을 수 있습니다.
- **적용 방안 (Hybrid 전략)**: 
  1. **초기 런칭(MVP)**: 개인 자격으로 가입하여 발급받은 파트너사의 **화이트 라벨(Iframe) 위젯**을 가져와 임베드합니다. (URL 파라미터로 "목적지=LBJ"만 넘겨주면 검색 결과가 바로 뜨게 할 수 있습니다.)
  2. **고도화**: 이를 통해 시장 반응과 트래픽 전환율을 먼저 테스트하고, 예약 볼륨이 커져 우리만의 독자적인 UI/UX가 절실해질 때 '사업자 전환 및 자체 API 연동 방식'으로 업그레이드하는 것을 추천합니다.

## 4. 🛠️ 개발팀(AI)에 전달해 주셔야 할 항목

가입 및 승인이 완료되면, 아래의 정보들을 취합하여 제공해 주시면 곧바로 프론트엔드 및 백엔드 개발에 착수할 수 있습니다.

1. **API Key 또는 위젯 코드 (Widget HTML/Iframe)**
   - API 연동을 하실 경우 API Key를, 화이트 라벨을 쓰실 경우 발급받은 위젯 소스코드를 전달해 주세요.
2. **API 공식 명세서 (API Documentation URL)**
   - 파트너사에서 제공하는 개발자용 가이드 문서 링크.
3. **제휴사 고유 파트너 ID (Affiliate ID)**
   - 예약 발생 시 우리 플랫폼의 실적으로 잡히기 위한 식별자입니다.
