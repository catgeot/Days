# 장소 카드 모바일 헤더 및 본문 UI 겹침 최적화 과제 (Phase 3)

본 문서는 장소 카드(`PlaceCard`)의 모바일 레이아웃(헤더 및 본문)에서 발생하는 겹침(Overlapping) 현상을 해결하고 공간 활용도를 극대화하기 위한 다음 세션 작업 계획입니다.

## 1. 현재 문제점 및 점검 결과
최근 2열 구조(Row 1: 제목/북마크, Row 2: 네비게이션 탭)로 확장된 헤더(`PlaceChatPanel`)의 높이 축소 및 본문 패딩(`pt-[96px]`)을 적용하였으나, 다음과 같은 문제가 잔존함:

1. **상부 겹침 잔존**: 갤러리(`PlaceGalleryView`)와 스마트 툴킷(`ToolkitTab`)의 상단(제목 영역)이 여전히 헤더 배경에 가려짐.
2. **과도한 헤더 공간 점유 (공간 낭비)**: 
   - 헤더의 솔리드/반투명 배경 공간이 불필요하게 큼.
   - 리뷰(`LogbookTab`) 탭 진입 시, **[장소 카드 메인 헤더] + [리뷰 고유 헤더]**가 화면의 50% 가량을 차지하여 실제 읽을거리(본문) 영역이 심각하게 협소해짐.
3. **네비게이션 버튼 분리에 따른 공간 낭비**: 기존에는 갤러리가 기본값이라 다른 탭(리뷰, 위키 등) 버튼만 있었고, 다른 탭으로 갔을 때 '갤러리 복귀'를 토글 방식으로 썼음. 현재 갤러리 탭 버튼이 독립적으로 한 자리를 차지해 향후 기능 추가 시 공간 부족 우려.

## 2. 향후 아키텍처 개선 계획 (Next Steps)

### 목표: "헤더의 배경 공간을 최소화하고 텍스트(본문) 가독 영역을 극대화하라"

#### A. 플로팅 탭(Floating Tabs) 구조 도입 검토
- **개념**: 갤러리, 리뷰, 위키, 영상 등의 2열 네비게이션 버튼을 메인 헤더 컨테이너(배경이 있는 박스)에서 분리.
- **구현**: 버튼들이 헤더 배경 영역 바깥(아래쪽)으로 빠져나와, 마치 **본문 위에 버튼만 둥둥 떠 있는 듯한(Floating) 효과**를 줌.
- **기대효과**: 어두운 반투명 배경(`bg-[#05070a]/90`)이 차지하는 절대적 높이가 Row 1(제목 영역) 수준으로 대폭 축소됨. 본문 렌더링 시작점이 위로 올라가 겹침 현상 원천 해결.

#### B. 갤러리 버튼 토글(Toggle) 롤백 검토
- 네비게이션 목록에서 '갤러리' 버튼을 상시 노출하는 대신, 갤러리가 아닌 뷰(`WIKI`, `LOGBOOK`, `VIDEO`)에 진입했을 때만 '갤러리 복귀' 버튼이 활성화되거나 기존 탭 영역을 대체하는 방식(Toggle)으로 되돌림.
- 가로 스크롤 공간을 절약하여 새로운 탭이나 기능 추가 대비.

#### C. `LogbookTab` (리뷰) 내부 헤더 압축 및 통합
- **문제**: "장소 리뷰 / 평점 / 전체보기 / 내 리뷰 / 작성하기"가 차지하는 `LogbookTab` 내부 헤더의 높이가 너무 큼.
- **해결 방안**:
  1. 리뷰 탭 내부 헤더의 패딩 및 폰트 크기 극대화 축소 (예: 2줄 차지하는 레이아웃을 1줄 좌우 배치로 밀집).
  2. 스크롤 시 내부 헤더가 사라지거나(Hide on scroll), 플로팅 버튼(우측 하단 FAB `+` 버튼 등)으로 "작성하기" 액션을 이관하여 상단 공간 확보.

#### D. 패딩(`pt`) 세밀화
- 헤더 배경이 줄어드는 A안 적용 시, 본문에 부여했던 `pt-[96px]` 패딩을 재계산하여 불필요하게 비어있는 상단 공간을 줄이고 읽기 영역 최적화.

## 3. 다음 세션 작업 지침
이 문서를 읽는 AI는 다음 세션 시작 시 **반드시 본 문서를 분석**하고, 모바일 스크린의 세로 공간(Vertical Real Estate)을 최대한 아끼는 UI/UX 관점에서 위 A, B, C, D 플랜을 순차적으로 구현해야 합니다. (기존 갤러리 동작과 스크롤 프리징 픽스 로직은 건드리지 않도록 주의)
